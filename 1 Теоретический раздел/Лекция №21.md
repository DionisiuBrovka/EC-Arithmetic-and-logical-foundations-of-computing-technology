# Триггеры

## Цели и задачи лекции
- Понять роль триггеров в цифровой логике и архитектуре ЭВМ.  
- Овладеть типами триггеров (RS, D, T, JK) и их схемами реализации.  
- Научиться проектировать простые цепочки триггеров и анализировать их поведение.  
- Приобрести навыки использования триггеров в ассемблерных программах и низкоуровневом программировании.

## Ключевые понятия
| Термин | Определение |
|--------|-------------|
| **Триггер** | Логический элемент, сохраняющий состояние в течение времени, реагирующий на входы. |
| **Состояние** | Значение, которое триггер хранит (0 или 1). |
| **Синхронный** | Триггер, изменяющий состояние только при импульсе тактового сигнала. |
| **Асинхронный** | Триггер, реагирующий мгновенно на изменение входов без тактового сигнала. |
| **Кодирование** | Способ представления информации (например, двоичное, Gray, один-из-нескольких). |
| **Flip‑flop** | Обобщённое название для большинства триггеров. |
| **Поршень** | Компонент, обеспечивающий переключение состояния в триггере. |
| **Параллельная загрузка** | Возможность одновременной загрузки данных во все триггеры цепочки. |
| **Каскадирование** | Соединение нескольких триггеров последовательно, передавая состояние. |
| **Кодовый регистр** | Последовательность из нескольких триггеров, используемая для хранения данных. |

## Основное содержание

### 1. История и роль триггеров (10 мин)
- Появление в 1940‑х, роль в ранних компьютерах.  
- Триггеры как фундамент памяти и счётчиков.  
- Сравнение с аналоговыми резистивными цепями.

### 2. Классификация триггеров (15 мин)
- **Асинхронные** (RS, JK) и их особенности.  
- **Синхронные** (D, T, JK) – защита от glitches, необходимость тактового сигнала.  
- Обсуждение преимуществ/недостатков каждой категории.

### 3. Реализация RS‑триггера (15 мин)
- Схема на NAND‑/NOR‑логике.  
```md
     +-----+      +-----+
S --|>   |------|>   |-- Q
     | N |      | N |
R --|>   |------|>   |-- Q̅
     +-----+      +-----+
```
- Анализ состояний: Set, Reset, Undefined.  
- Пример использования в схеме “триггер‑счётчик”.

### 4. D‑триггер (15 мин)
- Синхронный переход, одно входное значение.  
- Схема на NAND‑логике.  
```md
      +-----+      +-----+
D --->|>   |------|>   |-- Q
      | N |      | N |
CLK-->|>   |------|>   |-- Q̅
      +-----+      +-----+
```
- Пояснение “захвата” значения при фронте такта.  
- Примеры: регистра, буфер данных, временной промежуток.

### 5. T‑триггер и JK‑триггер (15 мин)
- **T‑триггер**: переключение при T=1.  
- **JK‑триггер**: объединение RS и T, устранение неопределённого состояния.  
- Схемы и таблица переходов.  
- Практический пример: 4‑битовый счётчик.

### 6. Секвенсоры и цепочки триггеров (10 мин)
- Каскадирование D‑триггеров для создания кодового регистра.  
- Параллельная загрузка и синхронные счётчики.  
- Пример: 8‑битный регистр с загрузкой и чтением.

### 7. Триггеры в ассемблере и низкоуровневом программировании (10 мин)
- Как операторы `SET`, `RESET`, `XCHG` влияют на триггерные состояния.  
- Пример кода на ассемблере для управления 4‑битовым регистром.  

```asm
; Пример загрузки значения 0xA в регистр R0
LOAD R0, #0xA
; Счётчик
INC R0
```

### 8. Практические упражнения (10 мин)
- Дизайн схемы 4‑битового делителя на T‑триггерах.  
- Сравнение работы RS‑триггера и D‑триггера при одновременной активации входов.

## Выводы по лекции
- Триггеры являются базовыми строительными блоками цифровых устройств, обеспечивая сохранение и управление состояниями.  
- RS, D, T, JK триггеры имеют различную семантику, но все они можно реализовать с помощью логических вентилей.  
- Понимание схем и таблиц переходов позволяет проектировать сложные счётчики, регистры и контроллеры.  
- Ассемблерные инструкции напрямую взаимодействуют с триггерными элементами, что важно для низкоуровневого программирования.

# Методы минимизации логических функций

## Цели и задачи лекции
1. Понять основные методы минимизации логических функций.  
2. Научиться применять Карно‑таблицы и алгоритм Quine–McCluskey.  
3. Освоить практические примеры минимизации для цифровых схем.  
4. Развить навыки анализа и оптимизации булевых выражений.

## Ключевые понятия
- **Булева функция** – отображение из набора переменных в {0,1}.  
- **Имплисит** – строка, где переменная может принимать 0 или 1 (знак “–”).  
- **Карта Карно** – таблица, упорядоченная по Gray‑коду, для визуального поиска групп.  
- **Критерий минимизации** – минимальное число логических элементов и/или минимальная сложность схемы.  
- **Алгоритм Quine–McCluskey** – таблицовый метод, автоматизирующий поиск минимальных имплиситов.  
- **Порция** – набор имплиситов, покрывающий все единицы функции.  
- **Минимальная порция** – порция с наименьшим количеством имплиситов или наименьшей суммой их размерностей.  
- **Функция вида SOP** – Sum‑of‑Products (сумма произведений).  
- **Функция вида POS** – Product‑of‑Sums (произведение сумм).  
- **Ключевой термин** – переменная, влияющая на значение функции.

## Основное содержание

### 1. Введение в минимизацию (10 мин)
- Почему минимизация важна: экономия логических элементов, уменьшение стоимости и времени работы схем.  
- Обзор методов: Карно‑таблица, Quine–McCluskey, алгоритмы Смит‑Смита, алгоритм Эпшира.

### 2. Карно‑таблица (25 мин)
#### 2.1 Построение таблицы
- Выбор порядка переменных по Gray‑коду.  
- Заполнение таблицы значениями функции.
```markdown
| AB\CD | 00 | 01 | 11 | 10 |
|-------|----|----|----|----|
| 00    | 0  | 1  | 1  | 0  |
| 01    | 1  | 1  | 0  | 0  |
| 11    | 0  | 0  | 1  | 1  |
| 10    | 0  | 1  | 0  | 0  |
```
#### 2.2 Поиск групп
- Круглые группы: 1, 2, 4, 8… элементов.  
- Правила: группы должны быть прямоугольниками, размеры – степени двойки.  
#### 2.3 Формирование имплиситов
- Из групп выделяем переменные, которые не меняются в группе.  
- Пример: группа из четырёх 1‑й в строках 01 и 11 → имплисит `B`.

### 3. Алгоритм Quine–McCluskey (30 мин)
#### 3.1 Инициализация
- Перевод таблицы Карно в набор строк с 1‑й.  
- Сортировка по количеству единиц.
#### 3.2 Слияние
- Сравнение строк с разницей в одной позиции → объединяем, ставим “–”.  
- Повторяем до тех пор, пока не останется объединений.
#### 3.3 Таблица имплиситов
- Создаём таблицу покрытия: строки – имплиситы, столбцы – исходные minterms.  
- Выбор минимальной порции: метод «прямой» (Prime Implicant Chart).
#### 3.4 Пример
```text
Minterms: 0,1,2,5,6,7,8,9,10,14
```
- Итоговый минимальный SOP: `F = A'B' + AC + BCD'`.

### 4. Практические примеры (15 мин)
#### 4.1 Минимизация 4‑переменной функции
```text
F(A,B,C,D) = Σ(0,2,3,6,7,8,10,11,12,14,15)
```
- Карно‑таблица → группы → имплиситы.  
- Итог: `F = A'D' + B'C + BC'D + BCD'`.
#### 4.2 Преобразование в POS
```text
F = Π(1,4,5,9,13,15)
```
- Карно‑таблица для отрицания → минимальный POS: `F = (A+B)(A+¬C)(¬B+¬D)`.

## Выводы по лекции
- Карно‑таблица удобна для функций с ≤4 переменными, обеспечивает быстрый визуальный поиск.  
- Алгоритм Quine–McCluskey автоматически обрабатывает функции с большим числом переменных, но требует более сложных вычислений.  
- Минимизация уменьшает количество логических элементов, повышая эффективность схем.  
- Практика с примерами закрепляет понимание и навыки применения методов.

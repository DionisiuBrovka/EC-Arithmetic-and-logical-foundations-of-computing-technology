# Основные архитектурно‑технические решения современных микропроцессоров

## Цели и задачи лекции
- Понять, как современные микропроцессоры реализуют основные функции ЦПУ.  
- Овладеть ключевыми архитектурными концепциями (pipelining, superscalar, out‑of‑order).  
- Изучить примеры реальных архитектур (Intel x86‑64, ARM, RISC‑V).  
- Научиться анализировать схемы и код, демонстрирующие работу этих решений.

## Ключевые понятия
| Термин | Определение |
|--------|-------------|
| **Pipeline** | Последовательность стадий обработки инструкции, позволяющая обрабатывать несколько инструкций одновременно. |
| **Superscalar** | Архитектура, позволяющая выполнять более одной инструкции за такт благодаря нескольким исполнительным блокам. |
| **Out‑of‑Order Execution** | Выполнение инструкций в порядке, отличном от их последовательности в программе, при этом сохраняется семантика. |
| **Cache** | Быстрый кэш памяти, уменьшающий задержки доступа к данным и инструкциям. |
| **Branch Prediction** | Предсказание ветвления, ускоряющее выполнение ветвящихся инструкций. |
| **Micro‑architecture** | Реальная реализация архитектуры в виде схемы, включая кэш, pipeline, контроллеры и т.п. |
| **Instruction Set Architecture (ISA)** | Набор инструкций и регистров, видимых программному обеспечению. |
| **Vector Processing Unit (VPU)** | Специализированный блок для параллельной обработки массивов данных. |
| **SIMD** | Спецификация инструкций, выполняющих однотипные операции над несколькими элементами данных одновременно. |
| **Hardware Multithreading (SMT)** | Параллельное выполнение нескольких потоков в одном процессоре. |

## Основное содержание

### 1. Архитектурные основы (20 мин)
- **ISA vs. Micro‑architecture**: разница и взаимосвязь.  
- Классификация микропроцессоров: **RISC** и **CISC**.  
- Эволюция архитектур: от 8‑битных микропроцессоров до современных многоядерных систем.

### 2. Pipeline и его стадии (15 мин)
- Стандартные стадии: IF, ID, EX, MEM, WB.  
- Пример простого 5‑стадийного pipeline.  
- **Бойки данных** и способы их устранения (forwarding, stalls).

```assembly
; Псевдо‑код 5‑стадийного pipeline
IF:  fetch instruction
ID:  read registers
EX:  execute ALU operation
MEM: access memory
WB:  write result back to register
```

### 3. Superscalar и параллелизм в исполнении (15 мин)
- Многоблочная архитектура.  
- Параллельное выполнение ALU, FPU, и других блоков.  
- Пример: два ALU‑блока на 64‑битном процессоре.

### 4. Out‑of‑Order Execution (20 мин)
- Источник проблемы: зависимость данных.  
- **Reorder Buffer (ROB)**, **Reservation Stations**.  
- Пример: порядок выполнения инструкций 1–5 в зависимости от доступности данных.

### 5. Кэш‑системы и память (10 мин)
- Уровни кэша L1, L2, L3.  
- Политики замены (LRU, FIFO).  
- Пример кода доступа к массиву и его кэш‑эффект.

```c
int arr[1000];
for (int i = 0; i < 1000; ++i)
    arr[i] = i * 2;   // последовательный доступ → кэш‑промежуток
```

### 6. Branch Prediction и ветвление (10 мин)
- Основные алгоритмы: static, dynamic, BTB, TAGE.  
- Как предсказание ветвления влияет на throughput.  
- Пример кода с ветвлением и таблицей предсказаний.

```assembly
; Псевдо‑код предсказания ветвления
BTB:  // Branch Target Buffer
if (prediction == TAKEN)
    jump to target
else
    continue sequentially
```

### 7. Векторные и SIMD‑устройства (10 мин)
- Основные инструкции SIMD (SSE, AVX, NEON).  
- Пример кода на C с использованием SIMD‑препроцессорных директив.  

```c
#include <immintrin.h>
void add_vectors(float *a, float *b, float *c) {
    __m256 va = _mm256_loadu_ps(a);
    __m256 vb = _mm256_loadu_ps(b);
    __m256 vc = _mm256_add_ps(va, vb);
    _mm256_storeu_ps(c, vc);
}
```

### 8. Многопоточность и SMT (5 мин)
- Параллельные потоки внутри одного ядра.  
- Пример: два потока, выполняющих независимые инструкции.  

```c
#pragma omp parallel
{
    /* Thread 1 */
    compute_task_A();
    /* Thread 2 */
    compute_task_B();
}
```

## Выводы по лекции
- Архитектура современного микропроцессора – сложная совокупность технологий, направленных на увеличение производительности.  
- Понимание pipeline, superscalar и out‑of‑order execution помогает объяснить, почему одновременное выполнение инструкций критично.  
- Кэш‑системы и branch prediction снижают задержки доступа к памяти и ветвлениям.  
- SIMD и VPU расширяют возможности для обработки больших массивов данных.  
- Многопоточность внутри ядра повышает эффективность использования ресурсов.

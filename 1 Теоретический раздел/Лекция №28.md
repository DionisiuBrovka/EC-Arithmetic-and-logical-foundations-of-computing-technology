# Архитектура микропроцессора и микропроцессорных систем

## Цели и задачи лекции  
1. Познакомить студентов с базовыми понятиями и структурой микропроцессора.  
2. Понять роль и взаимодействие основных блоков: ALU, регистров, шины, контроллеров.  
3. Рассмотреть принципы организации микропроцессорных систем: однопроцессорные и многопроцессорные.  
4. Научить анализировать простые схемы микропроцессорных блоков и разбираться в их работе на примерах.

## Ключевые понятия  
| Термин | Определение |
|--------|-------------|
| **Микропроцессор** | Центральный вычислительный блок, выполняющий инструкции программы. |
| **ALU (Arithmetic Logic Unit)** | Устройство, реализующее арифметические и логические операции. |
| **Регистры** | Быстрые ячейки памяти внутри микропроцессора, хранящие данные и адреса. |
| **Шина** | Канал передачи данных, адресов и управляющих сигналов между компонентами. |
| **Кластер** | Совокупность нескольких микропроцессоров, работающих совместно. |
| **Периферия** | Внешние устройства, подключаемые к микропроцессорной системе. |
| **Кэш** | Быстрая память, хранящая часто используемые данные и инструкции. |
| **Нормальный режим** | Стандартный режим работы процессора, где выполняются пользовательские программы. |
| **Режим прерывания** | Специальный режим, активируемый внешним сигналом прерывания. |
| **Мультиплексор** | Устройство, выбирающее один из нескольких входов для передачи на выход. |

## Основное содержание

### 1. История и развитие микропроцессоров (5 мин)  
- Появление 4004, 8080, 6502.  
- Переход от 8‑битных к 32‑битным и 64‑битным архитектурам.  
- Влияние архитектур на современную вычислительную технику.

### 2. Базовая архитектура одного микропроцессора (15 мин)  
#### 2.1. Структура регистрового файла  
- Регистр общего назначения (AX, BX, CX, DX).  
- Специальные регистры (IP, SP, BP, FLAGS).  

```assembly
; Пример использования регистров в x86
MOV AX, 0x1234
ADD AX, BX
```

#### 2.2. ALU и операции  
- Арифметические (сложение, вычитание, умножение, деление).  
- Логические (AND, OR, XOR, NOT).  
- Позиционные (сдвиги, вращения).  

#### 2.3. Шины и контроллеры  
- Адресная шина: 16‑, 32‑битные варианты.  
- Датовая шина: 8‑, 16‑, 32‑битные варианты.  
- Контроллер шины: управление доступом, тайминг.  

#### 2.4. Программный счетчик и циклы  
- Как работает IP (Instruction Pointer).  
- Пример цикла на ассемблере.  

```assembly
LOOP_START:
    DEC CX
    JNZ LOOP_START
```

### 3. Организация микропроцессорных систем (15 мин)  
#### 3.1. Однопроцессорные системы  
- Взаимодействие с памятью и периферией.  
- Пример схемы: процессор – контроллер памяти – RAM – периферия.  

#### 3.2. Многопроцессорные и многопоточность  
- Синхронизация процессоров.  
- Пул процессов, распределённый доступ к памяти.  

#### 3.3. Кэш и память уровня L1/L2  
- Принцип работы кэша.  
- Модели кэш-координации (MESI).  

#### 3.4. Периферийные устройства и контроллеры  
- UART, I2C, SPI, PCIe.  
- Пример подключения датчика к микроконтроллеру.  

```c
// Пример кода для чтения датчика по I2C
i2c_read(0x40, buffer, 2);
```

### 4. Принципы разработки микропроцессорных систем (10 мин)  
- Выбор архитектуры (RISC vs CISC).  
- Оптимизация энергопотребления.  
- Системы реального времени и их требования.

## Примеры и иллюстрации  
- **Схема 1**: 8‑битный процессор с 4‑битным ALU.  
- **Схема 2**: Блок памяти с кэшом L1 и L2.  
- **Код 1**: Простейшая программа на ассемблере, реализующая сложение двух чисел.  

```assembly
; Сложение двух чисел
MOV AL, 0x0A
MOV BL, 0x05
ADD AL, BL
```

## Выводы по лекции  
- Микропроцессор – ядро любой вычислительной системы, состоящее из ALU, регистров и шины.  
- Эффективность работы системы определяется как архитектурой микропроцессора, так и его интеграцией в систему (память, периферия, кэш).  
- Понимание структуры и взаимодействия блоков позволяет проектировать и оптимизировать микропроцессорные системы.

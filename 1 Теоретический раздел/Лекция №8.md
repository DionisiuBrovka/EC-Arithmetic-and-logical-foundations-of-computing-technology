# Арифметические действия над числами с фиксированной запятой

## Цели и задачи лекции
1. Познакомить студентов с представлением чисел с фиксированной запятой в компьютерах.  
2. Рассмотреть основные арифметические операции (сложение, вычитание, умножение, деление) над такими числами.  
3. Выяснить влияние округления, переноса и переполнения на результаты вычислений.  
4. На практике применить правила выполнения арифметики с фиксированной запятой в языке ассемблера.

## Ключевые понятия
- **Число с фиксированной запятой** – представление целой и дробной части через фиксированный набор битов.  
- **Разрядность** – общее число бит, используемых для хранения числа.  
- **Коэффициент** – множитель, определяющий масштаб (часто 2^n).  
- **Округление** – процедура выбора ближайшего представимого значения при потере точности.  
- **Перенос** – перенос разрядов в результате арифметической операции.  
- **Переполнение** – выход результата за диапазон доступных разрядов.  
- **Сигнатурный/беззнаковый режим** – определяет, как трактовать старший бит.  
- **Позиционирование запятой** – фиксированное местоположение разрядов, разделяющих целую и дробную части.  
- **Ширина дробной части** – число разрядов, отведённых под дробную часть.  
- **Кодирование** – способ хранения чисел (например, прямой, двухкомплементный, ULS).

## Основное содержание

### 1. Представление чисел с фиксированной запятой (15 мин)
- Формат Qm.n: m бит целой части, n бит дробной части, общая разрядность m+n.  
- Пример: Q8.8 – 8 бит целой, 8 бит дробной.  
- Расчёт диапазона: от –2^m до 2^m‑1/2^n.  
- Таблица примеров и диапазонов.

```text
Q4.4  |  Диапазон  |  Минимум   |  Максимум
------+------------+------------+-----------
-8.0  |  -8.0000   |  -8.0000   |   7.9375
```

### 2. Арифметика: сложение и вычитание (20 мин)
- Совместное выравнивание разрядов: при сложении Q8.8 + Q8.8 → Q9.8 (потенциальный перенос).  
- Алгоритм: сложить разряды, учесть перенос, проверить переполнение.  
- Пример кода на ассемблере (x86):

```asm
; Q8.8 сложение
mov ax, [num1]    ; 16‑битное значение Q8.8
add ax, [num2]    ; добавляем второй операнд
adc ah, 0         ; перенос в старший байт (если нужен)
; результат в ax
```

### 3. Умножение (20 мин)
- Результат имеет двойную ширину: Qm.n * Qm.n → Q2m.2n.  
- Пример: Q4.4 * Q4.4 → Q8.8.  
- Алгоритм: умножить целую и дробную части, затем сдвинуть результат вправо на n разрядов.  
- Пошаговый пример:

```text
  3.125 (Q4.4)   → 0011 0010
* 2.75  (Q4.4)   → 0010 1100
--------------------------------
  8.59375 (Q8.8) → 1000 1101 1001 0100
```

### 4. Деление (20 мин)
- Для точного результата делитель расширяется: Qm.n / Qm.n → Qm.n.  
- Алгоритм: сдвиг делимого влево на n разрядов, затем обычное целочисленное деление.  
- Пошаговый пример деления 5.5 / 1.25:

```text
  5.5  (Q4.4) → 0101 1000
÷ 1.25 (Q4.4) → 0001 0010
--------------------------------
  4.4  (Q4.4) → 0100 0110
```

### 5. Округление и ошибки округления (10 мин)
- Методы: «срезка», «округление к ближайшему» (round‑half‑even), «округление вверх/вниз».  
- Влияние на финансовые расчёты и графику.  
- Практический пример: суммирование 0.1 (Q8.8) десять раз → 1.0, но 0.1001… может привести к 0.9999.

### 6. Переполнение и защита (5 мин)
- Пороговые значения, проверка overflow флага.  
- Пример кода:

```asm
add ax, bx
jo  overflow_handler   ; Overflow flag set
```

## Примеры и иллюстрации
```text
Схема Q8.8:
[ 8 бит целой ][ 8 бит дробной ]
```

```asm
; Пример сложения с проверкой переполнения
mov ax, [a]        ; Q8.8
add ax, [b]        ; Q8.8
jno no_overflow
call overflow_handler
no_overflow:
; дальнейшая обработка
```

## Выводы по лекции
- Числа с фиксированной запятой позволяют эффективно хранить и обрабатывать вещественные данные в системах ограниченной вычислительной мощности.  
- Операции требуют учёта разрядности, переноса и округления.  
- Правильное применение алгоритмов и проверок гарантирует точность и надёжность вычислений.

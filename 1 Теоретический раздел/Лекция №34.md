# Организация обмена информацией

## Цели и задачи лекции
- Понять роль и принципы обмена данными в вычислительных системах.
- Изучить основные методы и протоколы передачи информации.
- Рассмотреть примеры реализации обмена на уровне микросхем и программного обеспечения.
- Научиться анализировать эффективность и надежность каналов связи.

## Ключевые понятия
- **Канал связи** – физический или логический путь для передачи данных.
- **Модуль обмена** – устройство, осуществляющее передачу и прием информации.
- **Протокол** – набор правил, определяющих формат и порядок передачи.
- **Пакет** – единица данных, передаваемая по сети.
- **Буфер** – временное хранение данных в процессе передачи.
- **Синхронизация** – согласование временных параметров между передающей и принимающей сторонами.
- **Шум** – несанкционированные изменения сигнала, влияющие на корректность передачи.
- **Кодировка** – преобразование данных в формат, пригодный для передачи.
- **Поток** – непрерывная последовательность данных.
- **Диапазон частот** – спектр частот, используемый для передачи сигнала.

### 3. Протоколы передачи данных (15 мин)
- **UART, SPI, I²C** – особенности, применение в микроконтроллерах.
- **TCP/IP** – модель OSI, стек протоколов, сегментация.
- **Алгоритмы коррекции ошибок** – CRC, Hamming, Reed–Solomon.

```c
// Пример расчёта CRC-16
uint16_t crc16(const uint8_t *data, size_t len) {
    uint16_t crc = 0xFFFF;
    while (len--) {
        crc ^= *data++ << 8;
        for (int i = 0; i < 8; i++)
            crc = (crc & 0x8000) ? (crc << 1) ^ 0x1021 : crc << 1;
    }
    return crc;
}
```

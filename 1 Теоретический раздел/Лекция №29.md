# Организация и программная модель однокристальных микропроцессоров

## Цели и задачи лекции
1. Понять основные принципы организации однокристальных микропроцессоров.  
2. Изучить программную модель и набор инструкций таких процессоров.  
3. Освоить примеры кода на ассемблере, демонстрирующие работу с регистровой архитектурой.  
4. Научиться анализировать влияние архитектурных особенностей на эффективность программ.

## Ключевые понятия
- **Микропроцессор** – интегральная схема, реализующая ЦПУ.  
- **Однокристальный микропроцессор** – микропроцессор, размещённый на одном кристалле, без внешних контроллеров.  
- **Регистр** – небольшая память внутри ЦПУ, доступная за один такт.  
- **ALU (Arithmetic Logic Unit)** – арифметико‑логическое устройство.  
- **Кэш** – быстрый буферный слой памяти.  
- **Pipeline** – последовательность этапов обработки инструкции.  
- **RISC** – Reduced Instruction Set Computer – архитектура с простыми, однорядными инструкциями.  
- **CISC** – Complex Instruction Set Computer – архитектура с сложными инструкциями.  
- **Microcode** – микрокод, реализующий сложные инструкции.  
- **Программная модель** – набор инструкций, регистров и правил работы процессора.

## Основное содержание

### 1. Архитектурные особенности однокристальных микропроцессоров (30 мин)
- **Платформы**: AVR, PIC, STM32 (ARM Cortex‑M), 8051, MIPS.  
- **Регистрный набор**: размер, типы (рабочие, управляющие).  
- **ALU**: арифметические и логические операции, флаги состояния.  
- **Память**: внутренняя флэш/RAM, сегментная адресация.  
- **Периферийные блоки**: таймеры, UART, SPI, I²C, PWM, ADC.  
- **Переходы и прерывания**: таблица переходов, обработка прерываний.  

### 2. Программная модель и набор инструкций (25 мин)
- **Модель команд**: однорядные инструкции, фиксированная длина, регистровый доступ.  
- **Синтаксис ассемблера**: директивы, операнды, псевдоинструкции.  
- **Классификация инструкций**: арифметические, логические, переходы, загрузка/сохранение.  
- **Флаги состояния**: Zero, Carry, Sign, Overflow.  

### 3. Пайплайнинг и микрокод (15 мин)
- Этапы исполнения: Fetch, Decode, Execute, Memory, Write‑back.  
- Примеры: 5‑этапный пайплайн ARM Cortex‑M.  
- Микрокод как слой абстракции: примеры микрокода для сложных инструкций 8051.  

### 4. Практические примеры кода (20 мин)
```asm
; Пример сложения двух чисел на AVR
    .section .text
    .global _start
_start:
    ldi r16, 0x12          ; загрузить первое число
    ldi r17, 0x34          ; загрузить второе число
    add r16, r17           ; r16 = r16 + r17
    out 0x00, r16          ; вывести результат на порт
    nop                    ; завершение
```

```asm
; Пример работы с прерываниями на STM32 (ARM Cortex‑M)
    .section .text
    .global _start
_start:
    ldr r0, =0x40021000    ; адрес регистра NVIC_ISER
    ldr r1, =0x00000001    ; включить прерывание
    str r1, [r0]           ; записать
    cpsie i                ; включить глобальные прерывания
    b .                    ; бесконечный цикл
```

### 5. Эффективность и оптимизация (10 мин)
- Минимизация количества инструкций.  
- Использование регистровых пар и многопоточности.  
- Сжатие кода: команды с короткими адресами.  
- Влияние кэша и пайплайна на задержку.  

## Выводы по лекции
- Однокристальные микропроцессоры объединяют вычислительную и периферийную функциональность в одном кристалле, что обеспечивает компактность и экономичность.  
- Программная модель определяется набором инструкций, регистров и правил работы с памятью.  
- Пайплайнинг повышает производительность, но требует осторожного управления данными и зависимостями.  
- Практические знания ассемблера позволяют писать эффективный код и понимать, как работают высокоуровневые программы.

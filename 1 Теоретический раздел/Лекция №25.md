# Лекция: Арифметико‑логические устройства

## Цели и задачи лекции
- Понять роль арифметико‑логических устройств (АЛУ) в архитектуре ЭВМ.
- Изучить основные арифметические и логические операции над числами в различных кодах.
- Познакомиться с архитектурой ALU, схемами выполнения операций и их реализацией в ассемблере.
- Научиться анализировать и проектировать простые арифметико‑логические схемы.

## Ключевые понятия
| Термин | Определение |
|--------|-------------|
| **АЛУ (ALU)** | Арифметико‑логическое устройство, выполняющее базовые арифметические и логические операции над данными. |
| **Код знака и дополнения** | Представление целых чисел, где один бит — знак, остальные — абсолютная величина. |
| **Двоичный код** | Представление чисел в системе счисления с основанием 2. |
| **Бинарный код** | Упрощённое название двоичного кода, обычно применяемое к цифровым данным. |
| **Код сдвига** | Операция, перемещающая биты влево или вправо, эквивалентная умножению/делению на степень 2. |
| **Тестирование знака (sign test)** | Проверка бита знака для определения результата арифметической операции. |
| **Операция с переносом (carry)** | Бит, указывающий на перенос из младшего разряда в более старый при сложении. |
| **Флаг переполнения (overflow)** | Указывает, что результат арифметической операции выходит за пределы представимого диапазона. |
| **Код Шеннона (Gray code)** | Код, в котором соседние значения отличаются ровно одним битом. |
| **Код Хаффмана** | Сжатый код, используемый в алгоритмах сжатия данных. |

## Основное содержание

### 1. Архитектура арифметико‑логического устройства (30 мин)
- Структура ALU: арифметический блок, логический блок, мультиплексоры, регистры.
- Принцип работы: выбор операции по сигналу команды, выполнение и выдача результата.
- Пример схемы ALU для 8‑битного процессора.

```text
+-----------------------------+
|          ALU                |
|  +-------+  +-------+       |
|  | ARITH |  | LOGIC |       |
|  +-------+  +-------+       |
|      |          |           |
|  +---------------+         |
|  |  MUX (выбор)  |         |
|  +---------------+         |
|          |                 |
|      +-------+             |
|      | OUT  |             |
|      +-------+             |
+-----------------------------+
```

### 2. Арифметические операции над числами в разных кодах (25 мин)
- Сложение и вычитание в двоичном коде.
- Представление отрицательных чисел: код знака и дополнения (1‑комплемент, 2‑комплемент).
- Пример: сложение 8‑битных чисел `01101010` и `11110100` в 2‑комплементе.

```asm
; 8‑битное сложение в 2‑комплементе
MOV AL, 0x6A        ; 01101010
MOV BL, 0xF4        ; 11110100 (–12)
ADD AL, BL          ; AL = 00111010 (58)
```

### 3. Логические операции и их применение (15 мин)
- AND, OR, XOR, NOT, NAND, NOR.
- Сравнение значений, проверка условий.
- Пример: проверка, является ли число чётным.

```asm
MOV AL, 0x13        ; 00010011
AND AL, 0x01        ; AL = 00000001 → число нечётное
```

### 4. Операции сдвига и циклические сдвиги (10 мин)
- Сдвиг влево/вправо, логический/арифметический сдвиг.
- Циклический сдвиг: сдвиг с сохранением старшего бита.
- Пример: умножение на 2 через сдвиг.

```asm
MOV AL, 0x0F        ; 00001111
SHL AL, 1           ; 00011110 (30)
```

### 5. Флаги и контрольные биты (10 мин)
- Флаги переноса, нуля, знака, переполнения.
- Как они используются в условных переходах.
- Пример: условный переход при переполнении.

```asm
ADD AL, BL
JNO no_overflow    ; переход, если нет переполнения
```

### 6. Примеры реализации простых арифметико‑логических задач в ассемблере (10 мин)
- Сложение двух массивов.
- Вычисление модулей разницы.
- Минимальный код программы, демонстрирующий работу ALU.

```asm
; Сложение двух массивов A и B длиной N
MOV CX, N
LOOP_START:
    MOV AL, [A + CX - 1]
    ADD AL, [B + CX - 1]
    MOV [C + CX - 1], AL
    LOOP LOOP_START
```

## Выводы по лекции
- Арифметико‑логические устройства являются ядром вычислительной техники, обеспечивая выполнение всех базовых операций.
- Понимание кодировки чисел и работы флагов позволяет эффективно разрабатывать и анализировать программы на низком уровне.
- Практические примеры в ассемблере демонстрируют, как теоретические понятия реализуются в реальном коде.

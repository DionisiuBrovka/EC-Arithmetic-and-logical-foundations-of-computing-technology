# Функционально полные системы

## Цели и задачи лекции
- Понять, что такое функционально полная система и зачем она нужна в вычислительной технике.
- Изучить примеры функционально полных и неполных систем булевых операций.
- Научиться строить схемы и коды, реализующие заданные логические функции.
- Оценить роль функционально полных систем в проектировании цифровых устройств и компиляторов.

## Ключевые понятия
| Термин | Определение |
|--------|-------------|
| **Булева функция** | Отображение из набора булевых переменных в {0,1}. |
| **Функционально полная система** | Набор логических операций, с помощью которых можно реализовать любую булеву функцию. |
| **Канонические формы** | Представления функций через конъюнкцию или дизъюнкцию однословных минтермов (CNF, DNF). |
| **Канонический набор** | Минимальный набор операций, из которого можно построить все булевы функции. |
| **Эйлерова схема** | Схема, в которой каждая вершина (операция) используется ровно один раз. |
| **Кодовое представление** | Реализация логической функции в виде последовательности инструкций ассемблера. |
| **Кодирование** | Процесс перевода логических выражений в аппаратные схемы. |
| **Постановка задачи «полнота»** | Проверка, что система может реализовать любую функцию. |
| **Теорема Куайна–МакЛафлина** | Формула для определения минимальной суммы импликаций. |
| **Карта Квайна** | Графическое представление булевой функции для упрощения. |

## Основное содержание

### 1. Введение (10 мин)
- История развития булевой алгебры и её роли в цифровой технике.
- Понятие «полнота» в контексте булевых функций.

### 2. Теоретические основы (20 мин)
#### 2.1 Определение функционально полной системы
- Формальное условие: наличие достаточного набора операций для построения любой функции.
- Пример: {AND, NOT} — полная система; {XOR, AND} — неполная.

#### 2.2 Критерии полноты
- Наличие операции отрицания и одной операцией, не являющейся линейной (не только XOR).
- Теорема: система полна, если она не включена в одну из семи «плохих» классов (константы, только отрицание, только дизъюнкция, только конъюнкция, линейные, одноуровневые, монотонные).

#### 2.3 Канонические формы и их роль в доказательстве полноты
- DNF и CNF как базовые представления.
- Пример преобразования функции в DNF.

### 3. Примеры функционально полных систем (25 мин)
#### 3.1 Система {AND, NOT}
- Построение OR через DeMorgan.
- Схема реализации функции `F(a,b,c) = (a ∧ b) ∨ c` в виде ассемблерного кода.

```asm
; Ассемблерный пример для 8051
MOV A, a
ANL A, b
MOV R0, A      ; R0 = a AND b
MOV A, c
ORL A, R0      ; A = (a AND b) OR c
```

#### 3.2 Система {NAND} (универсальная операция)
- Выражение всех базовых операций через NAND.
- Схема «побитовой» реализации XOR через NAND.

```verilog
module xor_via_nand(
    input a, b,
    output y
);
    wire n1, n2, n3;
    nand n1(a, b, n1);
    nand n2(a, n1, n2);
    nand n3(b, n1, n3);
    nand y(n2, n3, y);
endmodule
```

#### 3.3 Система {XOR, AND} (неполная)
- Демонстрация недостижения функции `AND` из XOR и AND.
- Иллюстрация ограничения: все функции, получаемые из XOR и AND, являются линейными по одной переменной.

### 4. Практическая часть: проектирование схемы (20 мин)
#### 4.1 Задача: реализовать функцию `G(a,b,c) = a ⊕ (b ∧ c)`
- Выбор минимального набора операций: {XOR, AND, NOT}.
- Схема в логическом редакторе (схема на 4 уровня логических вентилей).

#### 4.2 Кодирование в ассемблере
```asm
; 8051 код для G
MOV A, b
ANL A, c
MOV R1, A      ; R1 = b AND c
MOV A, a
XRL A, R1      ; A = a XOR (b AND c)
```

#### 4.3 Анализ производительности
- Сравнение количества вентилей и задержек для разных наборов операций.

### 5. Итоги и практические рекомендации (5 мин)
- Подчёркивание важности выбора функционально полной системы при проектировании микросхем.
- Рекомендация использовать NAND или NOR как базовые элементы для экономии ресурсов.

## Выводы по лекции
- Функционально полная система позволяет реализовать любую булеву функцию; это ключевой принцип цифровой схемотехники.
- Системы вида {AND, NOT} и {NAND} являются наиболее распространёнными и удобными в практических проектах.
- Понимание критериев полноты помогает быстро оценивать возможности набора операций и выбирать оптимальные схемы.

# Устройства управления

## Цели и задачи лекции  
1. Ознакомить студентов с принципами работы устройств управления в вычислительной технике.  
2. Показать связь между логическими схемами и архитектурой ЭВМ.  
3. Научить анализировать схемы управления и писать простые программы на ассемблере, использующие управляющие сигналы.  

## Ключевые понятия  
- **Устройство управления** – блок ЭВМ, генерирующий управляющие сигналы для остальных компонентов.  
- **Цикл обработки инструкции (fetch‑decode‑execute)** – последовательность операций, выполняемых процессором.  
- **Шина данных** – канал передачи цифровых данных между компонентами.  
- **Шина адреса** – канал передачи адреса памяти, указывающего, где хранится/читается информация.  
- **Порты ввода/вывода** – интерфейсы, позволяющие устройству управления взаимодействовать с внешними устройствами.  
- **Микрокод** – низкоуровневый код, реализующий инструкции процессора.  
- **Программно‑управляемая шина (bus)** – шина, на которой управляющие сигналы управляются программно.  
- **Сигнал «такт» (clock)** – синхронизационный сигнал, определяющий скорость выполнения операций.  
- **Регистры** – небольшие быстрые памяти внутри процессора.  

## Основное содержание  

### 1. Архитектура устройства управления (20 мин)  
- Структура: декодер инструкций, микросхема управления, контроллер шины.  
- Роль контроллера шины в распределении сигналов по шинам.  
- Пример схемы:  

```text
┌─────────────────────┐
│   Декодер инструкций │
├─────┬─────┬───────────┤
│     │     │           │
│  Сигналы управления  │
│     │     │           │
└─────┴─────┴─────┬─────┘
                    │
┌───────────────┐   │
│   Микросхема   │   │
│   управления   │   │
└──────┬────────┘   │
        │            │
┌───────┴────────────┴───────┐
│       Шина данных/адреса    │
└────────────────────────────┘
```

### 2. Цикл обработки инструкции (30 мин)  
- Фазы цикла: **fetch** – получение инструкции из памяти; **decode** – разбор операции; **execute** – выполнение.  
- Устройства, задействованные на каждой фазе: регистр PC, декодер, ALU, регистр IR.  
- Таблица управляющих сигналов для каждой фазы.  

```text
| Фаза   | Устройства              | Умолч. сигналы            |
|--------|-------------------------|----------------------------|
| fetch  | PC → адрес шины, RAM    | MemRead, IRWrite           |
| decode | IR → декодер, ALU      | Decoded, ALUControl        |
| execute| ALU, регистры, память  | ALUOut, MemWrite/Read      |
```

### 3. Микрокод и микропрограммы (25 мин)  
- Определение микрокода.  
- Формат микропрограммных инструкций (операнды, адреса).  
- Пример микропрограммы на простом микроархитектуре.  

```assembly
; Микропрограмма: загрузка и сложение
START:
    LOAD R0, [ADDR]     ; 01 00 10
    ADD  R1, R0, R2     ; 02 01 02
    STORE R1, [ADDR+4]  ; 03 01 14
    HALT                ; FF
```

### 4. Управление внешними устройствами (15 мин)  
- Порты ввода/вывода как расширение шины управления.  
- Пример программного контроля порта:  

```assembly
MOV  PORTA, 0xFF  ; установить все линии порта в 1
```

### 5. Практический пример: счётчик (10 мин)  
- Схема счётчика на 8‑битной шине.  
- Программа на ассемблере, реализующая счётчик.  

```assembly
ORG 0x00
START:
    MOV  COUNT, 0x00
LOOP:
    INC  COUNT
    JNZ  LOOP
    HLT
```

## Выводы по лекции  
- Устройства управления координируют работу всех компонентов ЭВМ.  
- Цикл fetch‑decode‑execute является ядром любой архитектуры.  
- Микрокод обеспечивает гибкость и расширяемость процессора.  
- Управление внешними устройствами расширяет функциональность системы.

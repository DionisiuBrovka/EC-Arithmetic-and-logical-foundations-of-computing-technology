# Система команд микропроцессоров, форматы команд, классификация команд, способы адресации

## Цели и задачи лекции
- Понять структуру и назначение команд микропроцессоров.  
- Овладеть классификацией команд по функциональным группам.  
- Изучить основные форматы команд и способы адресации.  
- Научиться анализировать и составлять простые ассемблерные программы.

## Ключевые понятия
| Термин | Определение |
|--------|-------------|
| **Микропроцессор** | Центральное устройство ЭВМ, выполняющее инструкции. |
| **Инструкция (команда)** | Операционная единица, кодируемая в машинном коде, определяющая действие. |
| **Формат команды** | Структура байтовой последовательности: коды операции, операнды, адреса. |
| **Классификация команд** | Деление команд по типу действия: арифметические, логические, переходы, ввод/вывод. |
| **Способы адресации** | Методы указания местоположения операндов: непосредственная, регистра, память, индексированная и т.д. |
| **Регистровый набор** | Коллекция регистров микропроцессора, используемых для хранения данных и адресов. |
| **Код операции (opcode)** | Числовой код, идентифицирующий тип команды. |
| **Параметр (operand)** | Значение, на котором выполняется операция. |
| **Промежуточный результат** | Значение, записанное в регистр ALU после выполнения команды. |
| **Поток выполнения** | Последовательность исполнения команд, управляемая указателями. |

## Основное содержание

### 1. Архитектура командной системы (15 мин)
- Общая схема микропроцессора: ALU, регистры, шина адресов/данных.  
- Пояснение роли командного блока (Control Unit).  
- Пример схемы типичного 8‑битного микропроцессора.

```text
+---------------------+
|   Control Unit      |
|  (Decoder + FSM)    |
+--------+------------+
         |
         v
+---------------------+
|          ALU        |
+--------+------------+
         |
         v
+---------------------+
|   Register File     |
+---------------------+
```

### 2. Форматы команд (20 мин)
- Стандартный формат: `[opcode][operand1][operand2]`.  
- Различные длины команд: однобайтовые, двухбайтовые, трехбайтовые.  
- Пример формата 8086: `[opcode][modrm]`.  
- Таблица типовых форматов для 8086/8088 и 8051.

| Формат | Длина | Компоненты |
|--------|-------|------------|
| 1‑байт | 8 бит | opcode |
| 2‑байт | 16 бит | opcode, адрес операнда |
| 3‑байт | 24 бит | opcode, регистр, адрес |
| 5‑байт | 40 бит | opcode, регистр, сегмент, offset |

### 3. Классификация команд (20 мин)
- **Арифметические**: ADD, SUB, MUL, DIV.  
- **Логические**: AND, OR, XOR, NOT.  
- **Перемещение**: MOV, XCHG.  
- **Переходы**: JMP, JZ, JNZ, CALL, RET.  
- **Ввод/вывод**: IN, OUT, PIO.  
- **Управляющие**: NOP, HLT, CLI, STI.  

Примеры кода (8051):

```asm
MOV A, #0x55      ; непосредственный перенос
ADD A, R2         ; арифметика
JZ  END           ; условный переход
```

### 4. Способы адресации (20 мин)
- **Непосредственная**: значение в команде.  
- **Регистровая**: операнд в регистре.  
- **Памятная**: прямой/потенциальный адрес в памяти.  
- **Индексированная**: адрес = регистр + смещение.  
- **Последовательная**: использование указателя (PC, SP).  

Таблица примеров адресации для 8086:

| Способ | Код | Пример |
|--------|-----|--------|
| Непр. | `MOV AL, 0x10` | Перенос 0x10 в AL |
| Регистр | `MOV AX, BX` | Перенос содержимого BX в AX |
| Память | `MOV AL, [1234h]` | Чтение из памяти |
| Индексированная | `MOV AL, [BX+SI]` | Чтение из адреса BX+SI |

### 5. Практическое применение (15 мин)
- Составление простого алгоритма: сложение двух чисел из памяти.  
- Разбор машинного кода.  
- Мини‑проект: написать программу, реализующую цикл с переходами.

```asm
ORG 0x0100
START:
    MOV AX, [NUM1]   ; загрузка первого числа
    ADD AX, [NUM2]   ; добавление второго
    MOV [RES], AX    ; запись результата
    HLT
NUM1 DW 0x0005
NUM2 DW 0x0007
RES  DW ?
```

## Примеры и иллюстрации
- График работы ALU при выполнении команды ADD.  
- Диаграмма переходов (FSM) для условного перехода JZ.

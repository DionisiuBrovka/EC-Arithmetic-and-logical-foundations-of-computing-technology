# Структура и типы ЭВМ

## Цели и задачи лекции
- Понять основные компоненты ЭВМ и их функции.
- Разобрать классификацию ЭВМ по архитектуре и назначению.
- Познакомиться с типами процессоров и их особенностями.
- Освоить примеры схем и кода, иллюстрирующие структуру ЭВМ.

## Ключевые понятия
1. **ЭВМ (Электронно-вычислительная машина)** – система, выполняющая арифметические и логические операции над данными.  
2. **Центральный процессор (CPU)** – ядро ЭВМ, осуществляющее обработку инструкций.  
3. **Память (RAM, ROM)** – устройства хранения данных и программ.  
4. **Устройства ввода/вывода (I/O)** – интерфейсы для взаимодействия с внешним миром.  
5. **Машинный код** – набор инструкций, понятных CPU.  
6. **Архитектура команд** – набор инструкций и регистров, определяющий поведение процессора.  
7. **Сегментация** – разделение памяти на логические сегменты.  
8. **Пайплайнинг** – параллельная обработка инструкций в разных стадиях.  
9. **Кэш** – быстрое временное хранилище данных.  
10. **RISC / CISC** – классы архитектур, отличающиеся сложностью инструкций.

## Основное содержание

### 1. Структура ЭВМ (15 мин)
- **Центральный блок**: CPU, кэш, регистры.  
- **Память**: иерархия (ROM, RAM, кэш, внешняя память).  
- **Переходные и I/O устройства**: шины, контроллеры, порты.  
- **Блок управления**: декодер инструкций, контроллеры тактирования.  

#### Схема типичной ЭВМ
```text
┌─────────────────────┐
│   Центральный блок  │
│  ┌───────┬─────────┐│
│  │ CPU  │ Cache   ││
│  └───────┴─────────┘│
└───────┬─────────────┘
        │
┌───────▼─────────────┐
│    Память (RAM)     │
└───────┬─────────────┘
        │
┌───────▼─────────────┐
│  Внешние устройства │
└─────────────────────┘
```

### 2. Типы ЭВМ по назначению (10 мин)
- **Системные ЭВМ** – компьютеры общего назначения (PC, сервера).  
- **Встроенные ЭВМ** – микроконтроллеры, PLC, устройства IoT.  
- **Мобильные ЭВМ** – смартфоны, планшеты.  
- **Суперкомпьютеры** – параллельные вычисления, научные исследования.  
- **Серверы** – хранение и обработка больших объемов данных.

### 3. Архитектурные модели процессоров (10 мин)
- **CISC** (Complex Instruction Set Computer) – сложные инструкции, меньше кода.  
- **RISC** (Reduced Instruction Set Computer) – простые инструкции, более быстрый цикл.  
- **VLIW** – Very Long Instruction Word, параллельная декодировка.  
- **ARM** – популярная RISC-архитектура для мобильных устройств.  

#### Пример инструкции в ARM (assembly)
```assembly
MOV R0, #10      ; загрузить 10 в регистр R0
ADD R1, R0, #5   ; R1 = R0 + 5
```

### 4. Пайплайнинг и кэш (10 мин)
- **Пайплайнинг**: 5‑стадийный процесс (IF, ID, EX, MEM, WB).  
- **Кэш L1, L2, L3**: уменьшение времени доступа к памяти.  

#### Пайплайн 5‑стадий
```text
|  IF  |  ID  |  EX  | MEM |  WB  |
|      |      |      |     |      |
|      |      |      |     |      |
```

### 5. Пример схемы микропроцессора (10 мин)
```text
┌─────────────────────┐
│  Устройство ввода    │
│  (порты, контроллеры)│
└───────┬──────────────┘
        │
┌───────▼──────────────┐
│   Центральный блок   │
│  (CPU, кэш, регистры)│
└───────┬──────────────┘
        │
┌───────▼──────────────┐
│   Память (RAM/ROM)   │
└───────┬──────────────┘
        │
┌───────▼──────────────┐
│   Устройства вывода   │
│  (порты, контроллеры)│
└──────────────────────┘
```

## Выводы по лекции
- ЭВМ состоит из CPU, памяти и I/O устройств, взаимодействующих по шинам.  
- Классификация по назначению и архитектуре помогает выбирать подходящие модели для задач.  
- Пайплайнинг и кэш повышают производительность, уменьшая время доступа к памяти.  
- Понимание архитектурных особенностей (CISC/RISC) необходимо для оптимизации программного кода.

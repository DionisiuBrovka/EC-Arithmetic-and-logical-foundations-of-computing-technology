# Общие принципы построения микропроцессорных систем

## Цели и задачи лекции
1. Понять, какие элементы образуют микропроцессорную систему и как они взаимодействуют.  
2. Изучить архитектурные особенности современных микропроцессоров.  
3. Научиться разрабатывать простые схемы микропроцессорных систем на основе описанных принципов.  
4. Применить знания о кодах и арифметике к проектированию и отладке микросхем.

## Ключевые понятия
- **Микропроцессор (CPU)** – центральный компонент, выполняющий инструкции.  
- **Память (RAM/ROM)** – область хранения данных и инструкций.  
- **Шина данных** – канал, по которому передаются байты/бит.  
- **Шина адресов** – канал, по которому указывается адрес памяти.  
- **Кэш** – быстрая память, ускоряющая доступ к часто используемым данным.  
- **Порты ввода/вывода (I/O)** – интерфейсы для связи с внешним миром.  
- **Периферийные устройства** – микросхемы, обрабатывающие специфические задачи (UART, SPI, I2C).  
- **Период тактирования** – частота, с которой работают все элементы.  
- **Инструкция** – двоичное слово, определяющее действие процессора.  
- **Кодирование чисел** – представление чисел в памяти (двоичный, восьмеричный, десятичный, шестнадцатеричный).

## Основное содержание

### 1. Архитектура микропроцессорной системы (15 мин)
- **Функциональные блоки**: ALU, регистры, контроллеры, шины.  
- **Классические схемы**: Harvard vs von Neumann.  
- **Пример схемы**: простая 8‑битная система с 16‑битными адресами.  

```text
  +-------------------+
  |      CPU          |
  |  +----+ +----+    |
  |  | ALU| |Regs|    |
  |  +----+ +----+    |
  +--------|----------+
           | 
  +--------|----------+
  |      RAM           |
  +-------------------+
```

### 2. Работа с памятью и шинами (10 мин)
- **Адресная и данные шины**: ширина, протокол передачи.  
- **Байтовый адрес** vs **битовый адрес**.  
- **Маскирование адресов** для разделения областей памяти.  

```assembly
; Пример обращения к памяти
LD   A, [0x2000]   ; загрузить байт из адреса 0x2000
```

### 3. Инструкционный набор и арифметика (10 мин)
- **Кодирование инструкций**: формат opcode + операнды.  
- **Базовые операции ALU**: сложение, вычитание, логические операции.  
- **Коды чисел**: представление целых и знаковых чисел в памяти.  

```text
Opcode: 0001  (ADD)
Operands:  A, B
Result: A ← A + B
```

### 4. Периферийные устройства и I/O (10 мин)
- **Портовый ввод/вывод**: порты 8‑битные, регистры управления.  
- **Протоколы**: UART, SPI, I2C.  
- **Обратная связь**: прерывания и DMA.  

```c
// Пример записи в порт UART
UART_TX = 0x55;
```

### 5. Тактирование и синхронизация (5 мин)
- **Кристальный генератор** и делители частоты.  
- **Синхронизация шины**: тактовые сигналы, задержки.  
- **Проблемы гонок** и способы их устранения.  

```text
CLK → CPU → RAM → I/O
```

## Примеры и иллюстрации
- **Схема 8‑битного процессора** с 256‑байтной памятью.  
- **Код на ассемблере** для суммирования двух чисел.  

```assembly
ORG 0x0000
START:
    LD   A, [0x00]    ; загрузить первый операнд
    ADD  A, [0x01]    ; добавить второй операнд
    ST   A, [0x02]    ; сохранить результат
    HLT
```

## Выводы по лекции
- Микропроцессорная система – это интегрированное сочетание CPU, памяти, шин и периферии.  
- Понимание архитектуры и кодирования инструкций позволяет проектировать эффективные и надёжные системы.  
- Практическое применение знаний в области кодирования чисел и работы с памятью критически важно для разработки встроенных систем.

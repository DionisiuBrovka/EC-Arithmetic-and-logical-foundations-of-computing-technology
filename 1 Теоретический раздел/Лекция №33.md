# Общие сведения о периферийных устройствах. Интерфейс. Анализ ОКР№2

## Цели и задачи лекции
- Познакомить студентов с классификацией и характеристиками периферийных устройств.
- Рассмотреть принципы взаимодействия устройств с центральным процессором через интерфейсы.
- Проанализировать конкретный пример ОКР№2, выделив основные элементы и способы диагностики.

## Ключевые понятия
- **Периферийное устройство** – аппаратное средство, подключаемое к ЭВМ и выполняющее ввод/вывод данных.  
- **Интерфейс** – совокупность средств и протоколов, обеспечивающих взаимодействие периферии и ядра.  
- **Память-перемещаемая** – тип памяти, требующей адресации и управления данными.  
- **Контроллер** – микросхема, управляющая работой периферийного устройства.  
- **Шина** – электрический канал передачи данных и команд между компонентами.  
- **ОКР (оперативно-контрольный регистр)** – регистр, в котором находятся статусные и управляющие биты.  
- **DMA (Direct Memory Access)** – метод передачи данных без участия процессора.  
- **IRQ (Interrupt Request)** – прерывание, сигнализирующее о необходимости обработки события.  
- **Буфер** – временное хранилище данных, позволяющее разгрузить процессор.  
- **Формат данных** – структура и кодировка информации, передаваемой через интерфейс.

## Основное содержание

### 1. Классификация периферийных устройств (10 мин)
- Вводные устройства (клавиатура, мышь, сканер).  
- Выводные устройства (монитор, принтер, аудио‑система).  
- Сетевые и коммуникационные устройства (сетевой адаптер, модем).  
- Хранилища (HDD, SSD, флеш‑накопители).  

### 2. Архитектура интерфейсов (12 мин)
- **Параллельные интерфейсы**: ISA, PCI, PCI‑Express.  
- **Последовательные интерфейсы**: UART, SPI, I²C, USB, SATA.  
- Схема обмена данными: адресация, чтение/запись, контроль ошибок.  

### 3. Контроллеры и шины (8 мин)
- Роль контроллера в управлении устройством.  
- Пример: контроллер клавиатуры (PS/2) – буфер, прерывание, статусные биты.  

### 4. ОКР№2 – анализ и диагностика (15 мин)
- Структура регистра ОКР№2 (битовый формат).  
- Пример кода для чтения состояния регистра в ассемблере.  
- Диагностические команды: `IN`, `OUT`, `TEST`.  
- Способы обработки ошибок (бит ошибок, флаг переполнения).  

```asm
; Пример чтения ОКР№2 на языке ассемблера x86
MOV   AX, 0x03F8      ; адрес порта COM1
IN    AL, AX           ; читаем статусный регистр
TEST  AL, 0x01         ; проверяем бит 0 – готовность к чтению
JZ    NoData           ; если нет данных, переходим к обработке
; далее читаем данные
```

### 5. Практические примеры работы с периферией (5 мин)
- Конфигурация UART для обмена данными.  
- Использование DMA для копирования больших блоков памяти.  

## Выводы по лекции
- Периферийные устройства существенно расширяют функциональность ЭВМ и требуют специализированных интерфейсов.  
- Контроллеры и шины играют ключевую роль в управлении и передаче данных.  
- ОКР№2 предоставляет важную информацию о состоянии устройства и позволяет быстро диагностировать ошибки.  
- Понимание структуры регистров и протоколов обмена критично для эффективной работы с периферией.

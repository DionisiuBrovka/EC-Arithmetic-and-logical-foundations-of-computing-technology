# Лабораторная работа № 10. Разработка и отладка программ с использованием арифметических команд

## Цель и задачи
1. Научиться разрабатывать программы на ассемблере, использующие арифметические команды ЭВМ.  
2. Освоить технику отладки и профилирования кода, анализируя работу регистров и памяти.  
3. Понять влияние разрядности и представления чисел (двоичного, восьмеричного, десятичного, шестнадцатичного) на арифметические операции.  
4. Развить навыки написания отчёта по лабораторной работе и оценивать корректность выполненных задач.

## Теоретическая часть
| Понятие | Определение |
|---------|-------------|
| Арифметическая команда | Инструкция ЭВМ, выполняющая арифметическую операцию над операндами (сложение, вычитание, умножение, деление). |
| Регистр | Быстрая ячейка памяти в процессоре, используемая для хранения операндов и результата. |
| Флаги состояния | Битовые флаги в регистре флагов, отражающие состояние выполнения операции (ZF, CF, OF, SF, PF). |
| Представление числа в двоичном коде | Число записывается в системе счисления с основанием 2, используется при работе с битами. |
| Представление числа в двоичном коде с дополнением до двух | Метод представления отрицательных чисел, где старший бит — знак, а остальные биты – дополнение. |
| База арифметических команд | Набор команд, доступных в конкретной архитектуре (x86, ARM, MIPS). |
| Сдвиг вправо/влево | Операции, перемещающие биты операнда на заданное число позиций, меняя при этом значение. |
| Деление с остатком | Операция, возвращающая как целую часть, так и остаток от деления. |
| Параллельность команд | Возможность выполнения нескольких арифметических команд одновременно в многопроцессорных системах. |

Формулы (для 16‑битного регистра):
- `sum = a + b`  
- `diff = a - b`  
- `prod = a * b`  
- `quot = a / b`  
- `mod = a % b`

## Ход выполнения работы
1. **Подготовка среды**  
   - Установить ассемблер NASM (Linux) или MASM (Windows).  
   - Создать проект `lab10.asm`.  
   - Проверить, что компилятор может собрать файл без ошибок:  
     ```bash
     nasm -f elf64 lab10.asm -o lab10.o
     ld lab10.o -o lab10
     ```

2. **Определение данных**  
   - Объявить переменные в сегменте `.data`:  
     ```asm
     section .data
         a dd 25
         b dd 17
         result dd 0
     ```

3. **Написание кода арифметических операций**  
   - В сегменте `.text` реализовать функции `add`, `sub`, `mul`, `div`.  
     ```asm
     section .text
     global _start

     _start:
         mov eax, [a]
         mov ebx, [b]
         add eax, ebx
         mov [result], eax   ; сохранение суммы

         mov eax, [a]
         sub eax, ebx
         mov [result], eax   ; вычитание

         mov eax, [a]
         imul ebx
         mov [result], eax   ; умножение

         mov eax, [a]
         cdq                ; подготовка к делению
         idiv ebx
         mov [result], eax   ; частное
         mov [quot], edx     ; остаток
     ```

4. **Отладка**  
   - Запустить программу в GDB:  
     ```bash
     gdb ./lab10
     (gdb) break _start
     (gdb) run
     (gdb) next
     (gdb) print $eax
     ```  
   - Проверить значения регистров после каждой операции и убедиться, что флаги (ZF, CF) выставлены корректно.

5. **Профилирование**  
   - Измерить время выполнения каждой функции, используя системный таймер `rdtsc`.  
   - Сравнить количество тактов для разных операций.

6. **Формирование отчёта** (см. ниже).

## Контрольные вопросы / задания для самопроверки
1. Что такое флаг CF и как он меняется при выполнении команды `add`?  
2. Как реализуется деление с остатком в x86?  
3. Объясните принцип работы знакового сдвига вправо.  
4. Напишите ассемблерный код, который умножает два 32‑битных числа и сохраняет результат в 64‑битном регистре.  
5. Какие флаги устанавливаются при выполнении команды `imul`?  
6. Как проверить, что число отрицательное, используя флаг SF?  
7. Опишите разницу между инструкциями `mov` и `lea` при работе с адресами.  
8. Как изменить представление числа из десятичного в двоичное в ассемблере?  
9. Что происходит, если при делении на ноль?  
10. Как измерить время выполнения программы без использования сторонних инструментов?

## Отчёт по работе
Структура отчёта должна включать:
1. **Титульный лист** – название работы, ФИО студента, группа, дата, преподаватель.  
2. **Цель и задачи** – кратко изложены выше.  
3. **Теоретическая часть** – ключевые определения, таблицы и формулы.  
4. **Описание программы** – структура кода, список функций, комментарии к ключевым строкам.  
5. **Результаты эксперимента** – таблица с данными входными операндами, результатами, временем выполнения и значениями флагов.  
6. **Анализ** – объяснение, почему результаты получены именно так, возможные ошибки и способы их исправления.  
7. **Выводы** – основные выводы и приобретённые навыки.  
8. **Приложения** – исходный код, скриншоты отладчика, лог-файлы.

## Критерии оценки
1. **Корректность программы** – отсутствие ошибок компиляции и логических ошибок.  
2. **Полнота отчёта** – соблюдение структуры и наличие всех разделов.  
3. **Аналитичность** – качественный анализ результатов и объяснение поведения флагов.  
4. **Техническая грамотность** – правильное использование команд, регистров и флагов.  
5. **Качество оформления** – соблюдение академического стиля и читаемость кода.

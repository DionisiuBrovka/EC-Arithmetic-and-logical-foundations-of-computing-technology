# Практическая работа № 8. Синтез логических схем в различных базисах

## Цель и задачи

1. Изучить принципы синтеза логических схем на основе различных логических базисов (AND‑OR‑NOT, NAND, NOR, XOR‑XNOR).
2. Научиться применять правила булевой алгебры для упрощения функций и выбора оптимального базиса.
3. Разработать схемы реализации заданных функций с минимальным числом вентилей и уровней.
4. Оценить влияние выбора базиса на площадь, скорость и энергопотребление схемы.

## Теоретическая часть

| Понятие | Определение |
|---------|-------------|
| **Логический базис** | Набор вентилей, из которых строятся все схемы. |
| **Постоянные** | Входы, принимающие фиксированные значения 0 или 1. |
| **Функция истинности** | Таблица, показывающая выходы для всех комбинаций входов. |
| **Упрощение функции** | Применение правил булевой алгебры для уменьшения числа терминов. |
| **Канонические формы** | Сумма произведений (SOP) и произведение сумм (POS). |
| **Базис NAND** | Любая логическая функция может быть реализована только с помощью вентилей NAND. |
| **Базис NOR** | Аналогично NAND, но с вентилями NOR. |
| **Минимизация схемы** | Поиск схемы с минимальным числом вентилей или уровней. |
| **Критерий Сатурна** | Минимальное число вентилей для реализации функции в заданном базисе. |
| **Критерий Леви** | Минимальное число уровней логической схемы. |

Формулы:
- `A' = A NAND A` (NOT из NAND)
- `A·B = (A NAND B) NAND (A NAND B)` (AND из NAND)
- `A + B = (A NAND A) NAND (B NAND B)` (OR из NAND)
- `A ⊕ B = ((A NAND B) NAND (A NAND B)) NAND ((A NAND A) NAND (B NAND B))` (XOR из NAND)

## Ход выполнения работы

1. **Анализ заданной функции**  
   Пояснение: Постройте таблицу истинности функции из условия.  
   Пример:  
   ```txt
   A B C | F
   0 0 0 | 0
   0 0 1 | 1
   ...
   ```  
   Убедитесь, что таблица корректна.

2. **Выбор канонической формы**  
   Пояснение: Запишите функцию как SOP и POS.  
   Пример:  
   ```txt
   F = Σ m(1,3,5,7) = A'BC + AB'C + ABC'
   ```

3. **Упрощение функции**  
   Пояснение: Примените правила булевой алгебры (дистрибутивность, идемпотентность, закон поглощения).  
   Пример:  
   ```txt
   F = A'BC + AB'C + ABC' → F = A'BC + AB'C + ABC' (не упрощается)
   ```

4. **Выбор базиса**  
   Пояснение: Определите, какой базис (AND‑OR‑NOT, NAND, NOR, XOR) наиболее подходит по количеству вентилей и уровней.  
   Пример:  
   - Для AND‑OR‑NOT: 3 AND + 3 OR + 3 NOT = 9 вентилей.  
   - Для NAND: 7 NAND.  
   - Для NOR: 7 NOR.

5. **Реализация схемы**  
   Пояснение: Постройте схему на выбранном базисе.  
   Пример кода для симуляции (Verilog, если применимо):  
   ```verilog
   module func(A,B,C, F);
     input A,B,C;
     output F;
     wire w1,w2,w3;
     nand n1 (w1, A, B);
     nand n2 (w2, B, C);
     nand n3 (w3, A, C);
     nand n4 (F, w1, w2, w3);
   endmodule
   ```

6. **Проверка схемы**  
   Пояснение: Сравните выход схемы со значениями из таблицы истинности.  
   Пример: Запустите симуляцию, убедитесь, что все комбинации совпадают.

7. **Анализ эффективности**  
   Пояснение: Подсчитайте количество вентилей, уровней и оцените влияние базиса на скорость.  
   Пример:  
   ```txt
   AND‑OR‑NOT: 9 вентилей, 3 уровня
   NAND: 7 вентилей, 2 уровня
   ```

8. **Документирование**  
   Пояснение: Составьте чертёж схемы, таблицу сравнения и выводы.  
   Пример: В чертеже указать типы вентилей, соединения и входы/выходы.

## Контрольные вопросы / задания для самопроверки

1. Что такое логический базис и какие существуют основные типы?  
2. Как построить таблицу истинности из логической функции?  
3. Опишите правила упрощения функций в булевой алгебре.  
4. Как преобразовать функцию, реализованную в базисе AND‑OR‑NOT, в базис NAND?  
5. Сколько вентилей понадобится для реализации функции `F = A·B + C'` в базисе NOR?  
6. Что такое критерий Сатурна и как он применяется при синтезе схем?  
7. В чем преимущества использования базиса NAND по сравнению с AND‑OR‑NOT?  
8. Как определить число уровней в логической схеме?  
9. Приведите пример реализации XOR только с NAND‑вентилями.  
10. Какие факторы влияют на выбор базиса при проектировании цифровых устройств?

## Критерии оценки

1. **Корректность схемы** – соответствие выходов таблице истинности.  
2. **Оптимальность** – минимальное число вентилей и уровней.  
3. **Качество отчёта** – полнота, логичность изложения, наличие схем и таблиц.  
4. **Теоретическая грамотность** – правильность примененных правил и формул.  
5. **Аналитический подход** – обоснованность выбора базиса и выводов.

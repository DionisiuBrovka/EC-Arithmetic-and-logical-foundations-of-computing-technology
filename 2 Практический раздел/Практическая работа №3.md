# Практическая работа № 3. Выполнение операции умножения двоичных чисел машинным методом

## Цель и задачи

1. Научиться реализовать умножение двоичных чисел с использованием машинного алгоритма «сдвиг‑и‑сложение» (shift‑and‑add).  
2. Понять роль разрядных сдвигов и побитовых операций в арифметике машинных данных.  
3. Развить навыки чтения и записи двоичных представлений чисел, включая знаковые и беззнаковые форматы.  
4. Оценить точность и эффективность реализованного алгоритма по сравнению с встроенной операцией умножения.

## Теоретическая часть

| Понятие | Определение / формула |
|---------|-----------------------|
| Двоичное число | Число, записываемое в системе счисления с основанием 2. |
| Беззнаковое представление | Число записывается только с помощью битов, без учёта знака. |
| Знаковое представление (дополнительный код) | Число с учётом знака, где старший бит – знак. |
| Сдвиг влево (`<<`) | Перемещение всех битов числа на заданное число позиций влево, добавляя нули справа. |
| Сдвиг вправо (`>>`) | Перемещение всех битов числа вправо, при знаковом сдвиге заполняя слева знаком. |
| Побитовое И (`&`) | Операция, результат которой – 1 только если оба бита 1. |
| Побитовое ИЛИ (`|`) | Результат – 1 если хотя бы один из битов 1. |
| Побитовый XOR (`^`) | Результат – 1 если биты различны. |
| Алгоритм «сдвиг‑и‑сложение» | Для каждого бита множителя, если он равен 1, к произведению прибавляется множимое, сдвинутое на позицию бита. |
| Размер регистра | Количество битов, на котором работает процессор (например, 32‑бит). |
| Переполнение | Случай, когда результат не помещается в заданный диапазон битов. |

Формула для беззнакового умножения по «сдвиг‑и‑сложению»:

\[
P = \sum_{i=0}^{n-1} (M_i \cdot (N << i))
\]

где \(M_i\) – \(i\)-й бит множителя, \(N\) – множимое, \(n\) – количество битов.

## Ход выполнения работы

1. **Подготовка входных данных**  
   - Выберите два 32‑битных беззнаковых числа `A` и `B`.  
   - Запишите их в двоичном виде.  
   - *Пример*:  
     ```text
     A = 0b00110101  (53 decimal)
     B = 0b00011010  (26 decimal)
     ```

2. **Инициализация переменных**  
   - Создайте переменную `result = 0`.  
   - Создайте копию множителя `B_copy = B`.  
   - *Пример на C*:  
     ```c
     uint32_t A = 53;
     uint32_t B = 26;
     uint32_t result = 0;
     ```

3. **Итерационный цикл по битам множителя**  
   - Для каждого бита `i` от 0 до 31:  
     - Если бит `i` множителя равен 1 (`(B >> i) & 1 == 1`), добавьте к `result` множимое, сдвинутое на `i` битов (`A << i`).  
     - В противном случае просто сдвиньте множимое на `i` битов и игнорируйте.  
   - *Пример на C*:  
     ```c
     for (int i = 0; i < 32; i++) {
         if ((B >> i) & 1) {
             result += (A << i);
         }
     }
     ```

4. **Обработка переполнения**  
   - После завершения цикла проверьте, не произошло ли переполнение: если результат не помещается в 32‑бит, отобразите предупреждение.  
   - *Пример на C*:  
     ```c
     if (result > UINT32_MAX) {
         printf("Переполнение!\n");
     }
     ```

5. **Проверка результата**  
   - Сравните `result` с результатом встроенного умножения `A * B`.  
   - Выведите оба результата в десятичном и двоичном виде.  
   - *Пример на C*:  
     ```c
     printf("Результат алгоритма: %u (0b%032b)\n", result, result);
     printf("Встроенное умножение: %u (0b%032b)\n", A * B, A * B);
     ```

6. **Анализ производительности**  
   - Замерьте время выполнения алгоритма и встроенного умножения при больших входных данных (например, 10 000 итераций).  
   - Сравните количество операций сдвига и сложения, использованных в вашем алгоритме, с количеством инструкций машинного умножения.  

## Контрольные вопросы / задания для самопроверки

1. Что происходит при сдвиге числа влево на 1 бит в беззнаковом представлении?  
2. Какой результат даст операция `A & B` при `A = 0b10101010` и `B = 0b11001100`?  
3. Напишите псевдокод алгоритма «сдвиг‑и‑сложение» для знакового умножения (с учётом знака).  
4. Как проверить, произошло ли переполнение при умножении двух 32‑битных чисел?  
5. Какие преимущества и недостатки имеет машинный метод «сдвиг‑и‑сложение» по сравнению с аппаратным умножением?  
6. Реализуйте функцию `multiply` на языке C, принимающую два `int32_t` и возвращающую `int64_t` (для хранения результата без потерь).  
7. Как изменится алгоритм, если использовать 16‑битные регистры?  
8. Поясните, почему в алгоритме «сдвиг‑и‑сложение» необходимо проверять каждый бит множителя.  
9. В каких случаях машинный метод «сдвиг‑и‑сложение» будет быстрее встроенного умножения?  
10. Напишите короткий тест, который проверяет корректность вашей реализации на случайных входных данных.

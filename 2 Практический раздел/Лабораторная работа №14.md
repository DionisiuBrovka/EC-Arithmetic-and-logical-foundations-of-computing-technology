# Лабораторная работа № 14  
**Исследование командного цикла микропроцессора в режиме моделирования на уровне микрокоманд**

## Цель и задачи
1. Познакомиться с принципами выполнения командного цикла микропроцессора.  
2. Изучить структуру и работу микрокоманд (фаза FETCH, DECODE, EXECUTE).  
3. Реализовать модель командного цикла на выбранном языке моделирования (Verilog/VHDL/С++).  
4. Проанализировать результаты моделирования и выявить возможные ошибки в реализации.  

## Теоретическая часть
1. **Микропроцессор** – устройство, исполняющее инструкции по последовательному чтению, декодированию и выполнению команд.  
2. **Командный цикл** – последовательность операций, выполняемых для каждой инструкции: FETCH → DECODE → EXECUTE → MEMORY ACCESS → WRITE BACK.  
3. **Микрокоманда** – элементарная операция, реализующая часть командного цикла (например, чтение из памяти, арифметический переход).  
4. **Регистры управления** – регистры, содержащие адрес текущей инструкции (PC), статус (PSW) и др.  
5. **Кодирование инструкций** – двоичное представление команд, определяемое архитектурой (например, 16‑битный формат с полем opcode и операндами).  
6. **Ключевые формулы**  
   - `PC_next = PC + 1` (если нет перехода).  
   - `IR = Memory[PC]` (загрузка инструкции).  
   - `ALU_result = ALU(operand1, operand2, opcode)` (вычисление).  
7. **Параллелизм микрокоманд** – одновременное выполнение нескольких микрокоманд в одном такте.  
8. **Тайминговая модель** – каждая микрокоманда занимает фиксированное число тактов, определяемое схемой.  
9. **Проверка корректности** – сравнение выходов модели с ожидаемыми результатами (таблицей истинности).  
10. **Синтез и верификация** – использование тестбенчей и средств симуляции (ModelSim, GTKWave).

## Ход выполнения работы
1. **Подготовка окружения**  
   - Установить симулятор (ModelSim/Verilog‑Sim) и текстовый редактор.  
   - Создать проект `MicrocycleLab`.  

2. **Определение формата инструкции**  
   ```verilog
   // 16‑битная инструкция
   // [15:12] – opcode, [11:0] – операнд
   reg [15:0] instruction;
   reg [3:0] opcode;
   reg [11:0] operand;
   assign {opcode, operand} = instruction;
   ```

3. **Моделирование фазы FETCH**  
   - Считывание инструкции из памяти по адресу `PC`.  
   - Увеличение `PC` на 1.  
   ```verilog
   always @(posedge clk) begin
     if (reset) PC <= 0;
     else PC <= PC + 1;
     IR <= Memory[PC];
   end
   ```

4. **Моделирование фазы DECODE**  
   - Извлечение `opcode` и `operand`.  
   - Выбор микрокоманды в зависимости от `opcode`.  

5. **Моделирование фазы EXECUTE**  
   - Реализация простейшего ALU (сложение, вычитание).  
   ```verilog
   always @(*) begin
     case (opcode)
       4'b0001: ALU_out = operand + regA; // ADD
       4'b0010: ALU_out = operand - regA; // SUB
       default: ALU_out = 0;
     endcase
   end
   ```

6. **Проверка результатов**  
   - Создать тестбенч, генерирующий набор инструкций.  
   - Сравнить `ALU_out` с ожидаемым значением.  

7. **Анализ таймингов**  
   - Сгенерировать график сигналов в GTKWave.  
   - Проверить, что все микрокоманды выполняются в пределах заданного такта.  

8. **Отладка**  
   - При несоответствии результатов исправить логику микрокоманд.  
   - Проверить регистры `PC`, `IR`, `PSW`.

# Лабораторная работа № 9  
Разработка и отладка программы с использованием различных форматов команд и методов адресации  

---

## Цель и задачи  

1. Освоить принципы работы с различными форматами команд в ассемблере.  
2. Научиться использовать основные методы адресации (непосредственная, непосредственная с оффсетом, регистровая, переменная, смещённая).  
3. Разработать и протестировать программу, демонстрирующую комбинированное применение команд разных форматов.  
4. Анализировать поведение программы в разных режимах работы (тестирование, отладка).  

---

## Теоретическая часть  

| Понятие | Определение |
|---|---|
| **Формат команды** | Структура команды, определяющая количество и тип операндов, размер кода и данных. |
| **Opcode (операционный код)** | Числовое представление операции, определяющее её семантику. |
| **Метод адресации** | Способ указания адреса операнда: непосредственный, переменный, регистровый, смещённый, индексированный, переменный с оффсетом. |
| **N‑битный регистр** | Регистры архитектуры, размер которых определяется числом битов (например, 16‑битный регистр AX). |
| **Псевдооперация** | Команда, реализуемая компилятором/ассемблером как последовательность реальных команд. |
| **Модель памяти** | Способ хранения данных и инструкций: линейный, сегментированный, страничный. |
| **Кодирование инструкций** | Преобразование логической операции в машинный код, учитывая формат и адресацию. |
| **Регистровая адресация** | Операнд находится в регистре. |
| **Памятная адресация** | Операнд находится в памяти по адресу, указанному в инструкции. |
| **Смещённая адресация** | Адрес операнда вычисляется как сумма базового адреса и смещения. |

## Ход выполнения работы  

1. **Подготовка среды**  
   * Откройте ассемблер (NASM, MASM, TASM) и эмулятор/дебаггер (GDB, OllyDbg).  
   * Создайте проект `lab9.asm`.  

2. **Определите набор команд**  
   * Выберите минимум 6 команд из семейства `ADD`, `SUB`, `MOV`, `CMP`, `JMP`, `CALL`.  
   * Для каждой команды определите формат (например, `ADD R1, R2` – регистровый, `MOV R1, 5` – непосредственный).  

3. **Разработайте программу**  
   ```asm
   ; lab9.asm
   section .data
       num1 db 10
       num2 db 20
   section .bss
       result resb 1

   section .text
       global _start
   _start:
       mov al, [num1]        ; регистровая адресация
       add al, [num2]        ; память‑память
       mov [result], al      ; память‑память
       mov eax, 1            ; системный вызов exit
       int 0x80
   ```
   * Комментарий к каждой инструкции: тип адресации, формат.  

4. **Сборка и линковка**  
   ```bash
   nasm -f elf32 lab9.asm -o lab9.o
   ld -m elf_i386 lab9.o -o lab9
   ```  

5. **Запуск и отладка**  
   * Запустите программу: `./lab9`.  
   * Используйте дебаггер: `gdb ./lab9`.  
   * Поставьте точки останова на каждую инструкцию и проверьте содержимое регистров и памяти.  

6. **Проверка разных форматов**  
   * Добавьте команду с непосредственным адресом: `mov al, 0xFF`.  
   * Добавьте команду со смещённой адресацией: `add al, [num1 + 1]`.  
   * Проверьте, что каждая команда корректно выполняется.  

7. **Сборка отчёта** (см. раздел «Отчёт по работе»).

# Лабораторная работа № 6  
Исследование работы регистров экспериментальным путем  

## Цель и задачи  
1. Изучить структуру и назначение регистров в современных микропроцессорах.  
2. Научиться считывать и записывать значения регистров при помощи ассемблерных инструкций.  
3. Проанализировать влияние изменений регистров на выполнение программы.  
4. Разработать навыки работы с отладчиком и средствами мониторинга регистров.  

## Теоретическая часть  
| Понятие | Определение | Пример/Формула |
|---------|-------------|----------------|
| Регистр | Быстрое внутреннее хранилище данных микропроцессора | R0, R1, R2… |
| Регистр общего назначения (GPR) | Хранит данные и адреса, доступные для программы | R0 = 0xFF |
| Регистр программного счётчика (PC) | Указывает адрес следующей инструкции | PC = 0x2000 |
| Регистр состояния (SR) | Хранит флаги (Zero, Carry, Overflow) | SR = 0b00001000 |
| Флаг Zero (Z) | Устанавливается, если результат операции = 0 | Z = 1, если R0 = 0 |
| Флаг Carry (C) | Устанавливается при переносе в арифметических операциях | C = 1 при 0xFF + 0x01 |
| Инструкция MOV | Перенос значения из одного регистра в другой | MOV R1, R0 |
| Инструкция ADD | Прибавление двух регистров | ADD R2, R1, R0 |
| Инструкция CMP | Сравнение двух регистров | CMP R1, R2 |
| Инструкция JMP | Безусловный переход | JMP 0x3000 |
| Инструкция JZ | Переход, если флаг Zero установлен | JZ 0x4000 |

## Ход выполнения работы  

1. **Подготовка среды**  
   - Открыть ассемблерный редактор (NASM, MASM) и отладчик (GDB, OllyDbg).  
   - Создать новый проект, задать сегмент данных `.data` и сегмент кода `.text`.  

2. **Инициализация регистров**  
   ```asm
   mov eax, 0x1234        ; загрузка значения в регистр EAX
   mov ebx, 0x5678        ; загрузка значения в регистр EBX
   mov ecx, 0x0           ; очистка регистра ECX
   ```  
   *Пояснение:* Мы устанавливаем известные значения, чтобы отслеживать их изменения.  

3. **Выполнение арифметической операции**  
   ```asm
   add eax, ebx           ; EAX = EAX + EBX
   ```  
   *Пример:* 0x1234 + 0x5678 = 0x68AC.  

4. **Сравнение результата с нулём**  
   ```asm
   cmp eax, 0
   je  label_zero         ; переход, если результат нулевой
   ```  
   *Пояснение:* Проверяем флаг Zero после операции.  

5. **Запись результата в память**  
   ```asm
   mov [result], eax
   ```  
   *Пример:* Переменная `result` в сегменте данных теперь содержит 0x68AC.  

6. **Мониторинг регистров в отладчике**  
   - Запустить программу в режиме отладки.  
   - Остановиться на точке останова после инструкции `add`.  
   - Проверить значения `EAX`, `EBX`, `ECX`, `EFLAGS`.  

7. **Изменение регистра состояния**  
   ```asm
   push eax
   pop eax                ; сохраняем и восстанавливаем EAX
   ```  
   *Пояснение:* Демонстрация работы стековых регистров.  

8. **Проверка переходов**  
   ```asm
   jmp after_cmp
   label_zero:
       mov edx, 1
   after_cmp:
   ```  
   *Пример:* Если результат был нулём, `EDX` установится в 1.  

9. **Вывод результата**  
   ```asm
   mov ebx, [result]
   ; вывести значение через системный вызов (Linux) или вызов API (Windows)
   ```  

10. **Сбор данных для отчёта**  
    - Снимки экрана отладчика с показом регистров до и после каждой инструкции.  
    - Таблица, показывающая изменения значений регистров.  

## Контрольные вопросы / задания для самопроверки  

1. Что хранит регистр состояния (SR) и какие флаги он может содержать?  
2. Какую инструкцию следует использовать для копирования значения одного регистра в другой?  
3. Как проверить, был ли выполнен перенос в результате сложения двух 16‑битных чисел?  
4. В чем разница между инструкциями `JMP` и `JZ`?  
5. Как можно вывести значение регистра в консоль при работе под Linux?  
6. Что произойдет, если выполнить инструкцию `MOV EAX, [EBX]`, где `EBX` содержит адрес памяти, но память не выделена?  
7. Как можно сохранить состояние регистров перед выполнением критической секции кода?  
8. Опишите, как отладчик позволяет наблюдать изменения регистров в реальном времени.  
9. Какие данные можно хранить в сегменте `.data` и как к ним обращаться из кода?  
10. Как изменится флаг Zero после выполнения инструкции `CMP EAX, EAX`?

# Практическая работа № 5  
**Выполнение арифметических операций сложения и вычитания чисел с плавающей запятой**

---

## Цель и задачи

1. Освоить правила представления чисел с плавающей запятой в IEEE‑754.  
2. Научиться выполнять сложение и вычитание чисел с плавающей запятой в ассемблере x86‑64.  
3. Разобрать влияние округления, разрядности и режима округления на результат.  
4. Научиться анализировать и интерпретировать полученные результаты с точки зрения точности вычислений.

---

## Теоретическая часть

| Понятие | Описание |
|---------|----------|
| **Число с плавающей запятой** | Представление вида ± m × 2^e, где m – мантисса, e – показатель. |
| **База** | 2 (для двоичного IEEE‑754). |
| **Формат single (32‑бит)** | 1 бит знака, 8 бит экспоненты, 23 бита мантиссы. |
| **Формат double (64‑бит)** | 1 бит знака, 11 бит экспоненты, 52 бита мантиссы. |
| **Показатель (exponent)** | Сдвинутый (bias) показатель: bias = 2^(k-1) – 1 (k – число бит экспоненты). |
| **Мантисса (mantissa)** | Нормализованная: первая единица всегда скрыта (implicit 1). |
| **Округление** | По правилам IEEE‑754: Round‑to‑Nearest‑Even (default), Round‑Toward‑Zero, Round‑Toward‑Positive, Round‑Toward‑Negative. |
| **Пограничные случаи** | NaN, +∞, –∞, +0, –0. |
| **Регистры x86‑64** | XMM0‑XMM15 – 128‑битные регистры для SIMD‑операций. |
| **Инструкции** | `addsd`, `subsd` – для double; `addss`, `subss` – для single. |
| **Порядок операций** | Вычисление сначала в регистре, затем сохранение в память. |

## Ход выполнения работы

1. **Подготовка среды.**  
   - Создайте проект в NASM (или MASM) с линковкой под Windows/Linux.  
   - Убедитесь, что линковщик поддерживает использование регистров XMM.  

2. **Определение переменных.**  
   ```asm
   section .data
       a dd 1.2345678      ; single precision
       b dd 2.3456789
       result dd 0.0
   section .bss
       res resd 1          ; резерв для double
   ```

3. **Загрузка значений в регистры XMM.**  
   ```asm
   movss   xmm0, dword [a]    ; загрузка a
   movss   xmm1, dword [b]    ; загрузка b
   ```

4. **Выполнение сложения.**  
   ```asm
   addss   xmm0, xmm1         ; xmm0 = a + b
   movss   dword [result], xmm0
   ```

5. **Выполнение вычитания.**  
   ```asm
   movss   xmm0, dword [a]
   movss   xmm1, dword [b]
   subss   xmm0, xmm1         ; xmm0 = a - b
   movss   dword [result], xmm0
   ```

6. **Вывод результатов.**  
   - Для печати используйте C‑стандартную библиотеку `printf` или системный вызов.  
   - Пример вызова из C:
     ```c
     extern float result;
     printf("a + b = %f\n", result);
     ```

7. **Анализ ошибок округления.**  
   - Повторите операции с числами, близкими к границе разрядности (например, 1.0e38 и 1.0e38).  
   - Сравните результат с вычислениями в Python/Matlab.  

8. **Тестирование разных режимов округления.**  
   ```asm
   ; установить режим округления Round‑Toward‑Zero
   mov eax, 0x0
   stmxcsr eax
   or eax, 0x00000100          ; флаг ROUND‑TO‑ZERO
   ldmxcsr eax
   ```

9. **Проверка специальных случаев.**  
   - Сложите +∞ и –∞, вычтите NaN.  
   - Сравните вывод с ожидаемыми результатами.

10. **Сборка и запуск.**  
    ```bash
    nasm -f elf64 -o fp.o fp.asm
    gcc -nostartfiles -o fp fp.o
    ./fp
    ```

## Контрольные вопросы / задания для самопроверки

1. Что такое «сдвинутый» показатель в IEEE‑754 и как вычисляется bias?  
2. В чем разница между инструкциями `addss` и `addsd`?  
3. Как установить режим округления Round‑Toward‑Zero в x86‑64?  
4. Что произойдет при сложении +∞ и –∞?  
5. Какой регистр используется для работы с числами двойной точности в x86‑64?  
6. Почему первая единица мантиссы скрыта в нормализованных числах?  
7. Как проверить, что результат операции равен NaN в ассемблере?  
8. Какие шаги необходимо предпринять, чтобы сравнить результат работы с результатом, полученным в Python?  
9. Что будет, если в переменной `a` хранится число 1.0e38, а в `b` – 1.0e-38, при сложении в формате single?  
10. Какие проблемы могут возникнуть при работе с числами, близкими к границе разрядности?

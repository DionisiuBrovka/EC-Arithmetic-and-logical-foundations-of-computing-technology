# Лабораторная работа № 2. Исследование работы шифратора экспериментальным путем

## Цель и задачи
1. Познакомиться с принципом работы шифратора (кодировщика) и его реализацией на уровне микропрограммирования.  
2. Научиться измерять скорость и точность работы шифратора при различных нагрузках.  
3. Развить навыки анализа экспериментальных данных и их представления в отчёте.

## Теоретическая часть
1. **Шифратор (кодировщик)** – устройство, преобразующее кодируемый сигнал в другой код, обычно для защиты от помех.  
2. **Кодирование** – процесс замены исходного сигнала на код, обеспечивающий более устойчивое передаточное свойство.  
3. **Код Хэмминга** – линейный блочный код, добавляющий контрольные биты для исправления ошибок.  
4. **Блок‑схема шифратора** – последовательность логических элементов (AND, OR, NOT, XOR) и триггеров.  
5. **Период выполнения** – время, за которое шифратор обрабатывает один набор входных данных.  
6. **Точность** – доля корректно закодированных данных в общей выборке.  
7. **Задержка (latency)** – время, прошедшее с момента подачи входа до появления выхода.  
8. **Плотность данных** – отношение количества кодируемых бит к количеству бит в выходном коде.  
9. **Параллельность** – одновременная обработка нескольких потоков данных.  
10. **Временная дискретизация** – измерение времени с помощью таймеров и счётчиков.

## Ход выполнения работы
1. **Подготовка окружения**  
   - Установить и настроить эмулятор микроконтроллера (например, STM32CubeIDE).  
   - Создать новый проект и подключить библиотеку HAL.  
   - Настроить таймер TIM3 для измерения периода выполнения шифратора.

2. **Разработка кода шифратора**  
   ```c
   // 4‑битовый шифратор с кодом Хэмминга (7‑битовый код)
   uint8_t hamming_encode(uint8_t data) {
       uint8_t d1 = (data >> 0) & 1;
       uint8_t d2 = (data >> 1) & 1;
       uint8_t d3 = (data >> 2) & 1;
       uint8_t d4 = (data >> 3) & 1;

       uint8_t p1 = d1 ^ d2 ^ d4; // контрольный бит 1
       uint8_t p2 = d1 ^ d3 ^ d4; // контрольный бит 2
       uint8_t p4 = d2 ^ d3 ^ d4; // контрольный бит 4

       return (p4 << 6) | (p2 << 5) | (p1 << 4) | (d4 << 3) |
              (d3 << 2) | (d2 << 1) | d1;
   }
   ```

3. **Интеграция таймера**  
   ```c
   void TIM3_Init(void) {
       __HAL_RCC_TIM3_CLK_ENABLE();
       TIM_HandleTypeDef htim3;
       htim3.Instance = TIM3;
       htim3.Init.Prescaler = 0;
       htim3.Init.CounterMode = TIM_COUNTERMODE_UP;
       htim3.Init.Period = 0xFFFF;
       HAL_TIM_Base_Init(&htim3);
       HAL_TIM_Base_Start(&htim3);
   }

   uint32_t measure_cycle(void (*func)(uint8_t), uint8_t data) {
       uint32_t start = __HAL_TIM_GET_COUNTER(&htim3);
       func(data);
       return __HAL_TIM_GET_COUNTER(&htim3) - start;
   }
   ```

4. **Экспериментальная проверка**  
   - Сгенерировать 1000 случайных 4‑битных чисел.  
   - Для каждого измерить время выполнения шифратора.  
   - Записать результаты в массив и вычислить среднее значение и дисперсию.  
   - Визуализировать данные (график «время vs. вход»).

5. **Анализ ошибок**  
   - Случайно ввести одну битовую ошибку в выходной код.  
   - Проверить корректность коррекции по коду Хэмминга.  
   - Подсчитать долю правильно исправленных ошибок.

6. **Подготовка отчёта** – см. раздел «Отчёт по работе».

## Контрольные вопросы / задания для самопроверки
1. Что такое шифратор и какие задачи он решает?  
2. Объясните принцип работы кода Хэмминга.  
3. Как измерить период выполнения шифратора с помощью таймера?  
4. Какие параметры характеризуют эффективность шифратора?  
5. Почему важно проверять коррекцию ошибок?  
6. Напишите формулу расчёта плотности данных для 4‑битового шифра с кодом Хэмминга.  
7. Какие дополнительные измерения можно добавить в эксперимент (например, энергопотребление)?  
8. Как изменится период работы шифратора при параллельной обработке 4 потоков?  
9. Какие ошибки могут возникнуть при неверной настройке таймера?  
10. Как бы вы улучшили реализацию шифратора для реального оборудования?

## Отчёт по работе
Отчёт должен содержать:
- **Введение** – цель и актуальность исследования.  
- **Теоретический раздел** – изложение ключевых понятий.  
- **Методика эксперимента** – подробное описание шагов, схемы подключения, используемые инструменты.  
- **Результаты** – таблицы, графики, расчёты средней задержки, дисперсии, точности.  
- **Обсуждение** – анализ полученных данных, возможные источники погрешностей.  
- **Выводы** – основные результаты и рекомендации.  
- **Приложения** – исходный код, скрипты, конфигурационные файлы.

## Критерии оценки
1. Полнота и корректность теоретической части.  
2. Точность и надёжность реализованного кода шифратора.  
3. Качество экспериментальных данных (объём, точность измерений).  
4. Структура и аргументация отчёта.  
5. Умение интерпретировать результаты и делать выводы.

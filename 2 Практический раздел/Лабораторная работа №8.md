# Лабораторная работа № 8. Анализ характеристик микропроцессора

## Цель и задачи
1. Ознакомиться с принципами измерения основных характеристик микропроцессора (частота, энергопотребление, тепловыделение, латентность операций).  
2. Научиться пользоваться программным и аппаратным инструментами для сбора данных о работе процессора.  
3. Проанализировать полученные данные и сделать выводы о работе и эффективности выбранной модели микропроцессора.  
4. Разработать краткий отчёт, включающий графики, таблицы и обоснованные выводы.

## Теоретическая часть
| Понятие | Определение | Формула / Команда |
|---------|-------------|-------------------|
| Частота (Hz) | Количество тактов в секунду | \(f = \frac{1}{T}\) |
| Такт | Минимальный период синхронизации | \(T = \frac{1}{f}\) |
| Потребляемая мощность | Средняя мощность, выделяемая процессором | \(P = V \cdot I\) |
| Тепловыделение | Энергия, превращающаяся в тепло | \(Q = P \cdot t\) |
| Задержка выполнения | Время, затраченное на выполнение операции | \(t_{\text{exec}} = \frac{1}{f_{\text{op}}}\) |
| Пакет данных | Набор инструкций, исполняемых без прерываний | – |
| Инструмент *Performance Counter* | Счетчик, встроенный в процессор для измерения событий | – |
| *Benchmark* | Набор тестов, измеряющих производительность | – |
| *Thermal Design Power (TDP)* | Максимальная тепловая нагрузка, которую система охлаждения должна уметь рассеять | – |

## Ход выполнения работы
1. **Подготовка оборудования**  
   - Подключить микропроцессор к тестовой плате.  
   - Установить датчики температуры (например, LM35) и токомеры.  
   - Подключить мультиметр к линиям питания для измерения напряжения и тока.  

2. **Измерение базовой частоты**  
   - Включить процессор и запустить таймер (или использовать встроенный *performance counter*).  
   - Записать значение тактовой частоты, полученное из системного реестра (на примере Intel: `rdtsc`).  

   ```c
   #include <stdio.h>
   #include <stdint.h>

   uint64_t rdtsc(void) {
       unsigned int lo, hi;
       __asm__ __volatile__ ("rdtsc" : "=a"(lo), "=d"(hi));
       return ((uint64_t)hi << 32) | lo;
   }

   int main(void) {
       uint64_t start = rdtsc();
       // короткая пауза
       uint64_t end = rdtsc();
       printf("ΔT = %llu cycles\n", end - start);
       return 0;
   }
   ```  

3. **Измерение энергопотребления**  
   - Включить мультиметр в цепь питания.  
   - Записать напряжение \(V\) и ток \(I\) при разных режимах работы (idle, load).  
   - Вычислить мощность \(P = V \cdot I\).  

4. **Измерение тепловыделения**  
   - Подключить датчик температуры к корпусу микропроцессора.  
   - Зафиксировать температуру \(T\) в режиме idle и под нагрузкой.  
   - Рассчитать тепловыделение \(Q = P \cdot t\) за заданный интервал времени \(t\).  

5. **Измерение латентности инструкций**  
   - Написать простую программу, выполняющую серию инструкций (например, `ADD`, `SUB`, `MUL`).  
   - Считать время выполнения каждой инструкции с помощью *high‑resolution timer*.  

   ```assembly
   ; Пример на x86‑64
   section .text
   global _start
   _start:
       mov rax, 5
       mov rbx, 3
       add rax, rbx          ; ADD
       sub rax, rbx          ; SUB
       imul rax, rbx         ; MUL
       ; завершить
       mov rax, 60
       xor rdi, rdi
       syscall
   ```  

6. **Анализ результатов**  
   - Составить таблицу с измеренными величинами.  
   - Построить графики зависимости мощности от частоты, температуры от нагрузки и т.д.  
   - Сравнить полученные значения с технической документацией (datasheet) микропроцессора.  

7. **Подготовка отчёта** (см. раздел «Отчёт по работе»).

## Контрольные вопросы / задания для самопроверки
1. Что такое тактовая частота и как она измеряется?  
2. Как рассчитывается потребляемая мощность микропроцессора?  
3. Что измеряет *performance counter* и зачем он нужен?  
4. Какое влияние имеет частота на энергопотребление?  
5. Опишите процедуру измерения температуры микропроцессора в лабораторных условиях.  
6. Как сравнить измеренные данные с официальной спецификацией?  
7. Почему важен анализ латентности отдельных инструкций?  
8. Какие факторы могут влиять на точность измерений температуры?  
9. Какое оборудование необходимо для измерения тока питания процессора?  
10. Какие выводы можно сделать, если измеренная температура превышает TDP?

## Отчёт по работе
Отчёт должен содержать:
1. **Титульный лист** – название работы, ФИО студента, номер группы, преподаватель, дата сдачи.  
2. **Введение** – цель лабораторной, краткое описание выбранного микропроцессора.  
3. **Теоретическая часть** – ключевые определения и формулы (таблица из раздела «Теоретическая часть»).  
4. **Методика измерений** – подробное описание оборудования, схемы подключения, используемые программы и коды.  
5. **Результаты** – таблицы с измеренными значениями, графики (частота‑мощность, температура‑нагрузка, латентность‑инструкция).  
6. **Анализ и обсуждение** – сравнение с данными из datasheet, объяснение расхождений, возможные источники погрешностей.  
7. **Выводы** – основные результаты, оценка эффективности микропроцессора, рекомендации по оптимизации.  
8. **Список использованных источников** – литература и стандарты.  
9. **Приложения** – исходные коды, скриншоты графиков, схемы подключения.

## Критерии оценки
1. Полнота и корректность отчёта (30 %).  
2. Точность и воспроизводимость измерений (25 %).  
3. Глубина анализа и обоснованные выводы (25 %).  
4. Чёткость и профессионализм оформления (10 %).  
5. Соблюдение требований к коду и комментариям (10 %).

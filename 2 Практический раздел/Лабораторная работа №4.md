# Лабораторная работа № 4. Исследование работы сумматора экспериментальным путем

## Цель и задачи  
1. Ознакомиться с принципами работы одноразрядного и многоразрядного сумматора.  
2. Научиться моделировать сумматор в логическом симуляторе и проверять его корректность.  
3. Разработать и протестировать простую программу на ассемблере, реализующую сложение двух 8‑битных чисел.  
4. Проанализировать влияние переноса (carry) на результаты сложения и построить таблицу истинности сумматора.  

## Теоретическая часть  

| Понятие | Определение / формула |
|---------|-----------------------|
| Сумматор (full adder) | Логический элемент, принимающий три входа (A, B, Cin) и выдающий два выхода (S, Cout). |
| Сложение без переноса | \(S = A \oplus B\) |
| Перенос (carry‑out) | \(C_{\text{out}} = (A \land B) \lor (Cin \land (A \oplus B))\) |
| Многоразрядный сумматор | Последовательное соединение одноразрядных сумматоров, где \(C_{\text{out}}\) предыдущего становится \(Cin\) следующего. |
| Таблица истинности одноразрядного сумматора | 8 строк, перечисляющих все комбинации A, B, Cin и соответствующие S, Cout. |
| Ассемблерный оператор `ADD` | Инструкция сложения двух регистров (или регистра и памяти) с автоматическим переносом в флаг Carry. |
| Кодирование чисел | Двоичный, восьмнадцатичный, десятичный. Для сумматора используются двоичные представления. |
| Флаги арифметики | Zero, Sign, Overflow, Carry. Они используются для контроля результата. |
| Синтаксис логических вентилей | `AND`, `OR`, `XOR`, `NOT`. |
| Симулятор логических схем | Программа (например, Logisim, Multisim), позволяющая построить схему и наблюдать выходы. |

## Ход выполнения работы  

1. **Подготовка схемы одноразрядного сумматора**  
   - Нарисуйте схему из вентилей `XOR`, `AND`, `OR`.  
   - Входы: `A`, `B`, `Cin`. Выходы: `S`, `Cout`.  
   - Укажите формулы (см. таблицу истинности).  

2. **Построение таблицы истинности**  
   - В таблице перечислите все 8 комбинаций входов.  
   - Заполните столбцы `S` и `Cout` по формулам.  
   - Проверьте, что таблица соответствует схеме.  

3. **Моделирование в логическом симуляторе**  
   - Создайте проект в Logisim.  
   - Вставьте вентиль `XOR` для вычисления `S`.  
   - Вставьте вентиль `AND` для вычисления промежуточных сигналов.  
   - Вставьте вентиль `OR` для получения `Cout`.  
   - Подключите входы и проверьте все комбинации, сравнивая с таблицей.  

4. **Построение 8‑разрядного сумматора**  
   - Создайте цепочку из 8 одноразрядных сумматоров, соединяя `Cout` предыдущего с `Cin` следующего.  
   - Добавьте начальный `Cin` (обычно 0).  
   - Проверьте работу схемы, вводя случайные 8‑битные числа.  

5. **Разработка ассемблерной программы**  
   ```asm
   ; Ассемблерная программа сложения двух 8‑битных чисел
   ; Регистр AX хранит первое число
   ; Регистр BX хранит второе число
   ; Результат (с переносом) сохраняется в CX

       MOV  AX, 0x3A      ; 00111010b
       MOV  BX, 0x15      ; 00010101b
       ADD  AX, BX        ; AX = AX + BX, флаг Carry выставлен при переносе
       MOV  CX, AX        ; сохраняем результат
   ```
   - Запустите программу в эмуляторе (например, DOSBox) и проверьте содержимое регистров.  
   - Сравните результат с тем, что предсказано таблицей истинности.  

6. **Экспериментальная проверка переноса**  
   - Введите пары чисел, при которых ожидается перенос (например, 200 + 100).  
   - Отслеживайте флаг Carry и значение результата.  
   - Запишите наблюдения в таблицу.  

7. **Анализ ошибок**  
   - Попробуйте ввести неверные значения (не 8‑битные).  
   - Опишите, как система обрабатывает такие случаи (обрезка, предупреждение).  

## Контрольные вопросы / задания для самопроверки  

1. Что делает вентиль `XOR` в сумматоре?  
2. Напишите формулу для `Cout` одноразрядного сумматора.  
3. Сколько строк в таблице истинности одноразрядного сумматора?  
4. Какой регистр в ассемблере хранит результат сложения, если использовать инструкцию `ADD AX, BX`?  
5. При каком сумме двух 8‑битных чисел флаг Carry будет установлен?  
6. Как изменить схему, чтобы получить сумматор без переноса?  
7. Что произойдет, если в один из входов подать значение, не являющееся 0 или 1?  
8. Как проверить корректность работы 8‑разрядного сумматора в симуляторе?  
9. В чем разница между флагом Carry и флагом Overflow?  
10. Опишите, как можно использовать таблицу истинности для проверки схемы.  

## Отчёт по работе  

Структура отчёта должна включать:  

1. **Введение** – цель лабораторной работы, краткое описание задачи.  
2. **Теоретическая часть** – перечисление ключевых понятий и формул.  
3. **Ход выполнения** – пошаговый разбор действий, схемы (с рисунками, если возможно), таблицы истинности.  
4. **Результаты** – таблица сравнения результатов симуляции и ассемблерной программы.  
5. **Анализ** – выводы о работе сумматора, влияние переноса, возможные ошибки.  
6. **Выводы** – обобщение полученных знаний, практическая ценность.  
7. **Приложения** – схемы, исходный код, скриншоты симуляции.

# Практическая работа № 4. Выполнение операции деления двоичных чисел машинным методом

## Цель и задачи
1. Научиться реализовывать деление двоичных чисел машинным методом в ассемблере.
2. Разобрать алгоритм деления с использованием смещения и вычитания.
3. Освоить работу с регистрами и флагами процессора при реализации арифметических операций.
4. Проверить корректность результата через ручной расчёт и автоматический тест.

## Теоретическая часть
1. **Машинный метод деления** – последовательное смещение делителя и вычитание из остатка.
2. **Регистры** – `AX`, `DX` (для 16‑битных операций), `EAX`, `EDX` (32‑битные).
3. **Флаги** – `CF` (carry), `OF` (overflow), `ZF` (zero), `SF` (sign).
4. **Команда `SHL`** – логическое сдвиг влево.
5. **Команда `SUB`** – вычитание с флагом переноса.
6. **Команда `CMP`** – сравнение (производит вычитание без изменения регистров).
7. **Команда `JNC`** – переход, если `CF` = 0 (нет переноса).
8. **Команда `INC`** – инкремент.
9. **Алгоритм деления**:
   - Инициализировать остаток `R = 0`.
   - Для i = n‑1 … 0:
     - Сдвинуть `R` влево на 1 бит, добавить очередной бит делимого.
     - Если `R ≥ D` (делитель), вычесть `D` из `R` и установить `Q[i] = 1`, иначе `Q[i] = 0`.
10. **Проверка** – `Q * D + R = N` (где `N` – делимое).

## Ход выполнения работы
1. **Подготовка среды**  
   - Открыть ассемблерный проект (MASM, NASM, TASM).  
   - Создать файл `div.asm`.

2. **Определение данных**  
   ```asm
   .data
       dividend  DWORD 0xABCD1234   ; 32‑битное делимое
       divisor   DWORD 0x00001234   ; 32‑битный делитель
       quotient  DWORD 0            ; результат деления
       remainder DWORD 0            ; остаток
   ```

3. **Инициализация**  
   ```asm
   .code
   start:
       mov     EAX, dividend      ; загружаем делимое
       mov     ECX, divisor       ; загружаем делитель
       xor     EDX, EDX           ; EDX будет хранить остаток (R)
       mov     EBX, 32            ; количество битов
   ```

4. **Основной цикл**  
   ```asm
   div_loop:
       shl     EDX, 1             ; R = R << 1
       rcr     EAX, 1             ; перенос бита из делимого в R
       cmp     EDX, ECX           ; сравниваем R и D
       jnc     sub_divisor        ; если R >= D, вычитаем
       jmp     next_bit
   sub_divisor:
       sub     EDX, ECX           ; R = R - D
       or      quotient, 1 << (EBX-1) ; устанавливаем бит в делителе
   next_bit:
       dec     EBX
       jnz     div_loop
   ```

5. **Сохранение результата**  
   ```asm
   mov     remainder, EDX
   mov     quotient, EAX
   ```

6. **Проверка корректности**  
   ```asm
   mul     ECX                 ; EAX:EAX = Q * D
   add     EAX, remainder      ; EAX = Q*D + R
   cmp     EAX, dividend
   je      success
   ```

7. **Выход из программы**  
   ```asm
   success:
       mov     AH, 4Ch
       int     21h
   ```

## Контрольные вопросы / задания для самопроверки
1. Что хранит регистр `EDX` в процессе выполнения алгоритма?  
2. Почему в цикле используется команда `rcr` вместо `shr`?  
3. Какой флаг проверяется в условном переходе `jnc` и почему?  
4. Как изменить программу для работы с 16‑битными числами?  
5. Поясните, что делает команда `or quotient, 1 << (EBX-1)`.  
6. Как изменить код, чтобы выводить результат в консоль?  
7. Почему после завершения цикла в регистре `EAX` находится делитель?  
8. Какие ошибки могут возникнуть, если делитель равен нулю?  
9. Как проверить, что остаток действительно меньше делителя?  
10. Как изменить программу для работы с отрицательными числами (два's complement)?

# Лабораторная работа № 11  
Разработка и отладка программ с использованием команд пересылки и передачи управления  

## Цель и задачи  
1. Изучить семантику команд переходов и вызовов в ассемблере.  
2. Научиться проектировать и реализовывать программы, использующие условные и безусловные переходы, а также процедуры и функции.  
3. Научиться отлаживать такие программы, анализируя таблицу регистра и стек.  
4. Развить умения документировать код и выводить результаты работы в виде отчёта.  

## Теоретическая часть  

| Понятие | Определение | Пример |
|--------|-------------|--------|
| **Пересылка (jump)** | Необусловленный переход к указанному адресу. | `JMP label` |
| **Условный переход** | Переход, если заданное условие истинно. | `JE label` – перейти, если Zero Flag = 1 |
| **Вызов процедуры (call)** | Сохраняет адрес возврата и переходит к процедуре. | `CALL sub` |
| **Возврат (ret)** | Возвращает управление в точку после `CALL`. | `RET` |
| **Стек** | Структура LIFO, в которой хранятся адреса возврата и локальные данные. | `push ebp / mov ebp, esp` |
| **Флаги процессора** | Флаги состояния, используемые в условных переходах. | CF, ZF, SF, OF |
| **Код возврата** | Значение, возвращаемое из функции. | `MOV eax, result / RET` |
| **Параметры функции** | Передаются через стек или регистры. | `push arg / CALL func` |
| **Семантика `LOOP`** | Уменьшает CX/ECX и переходит, если результат не ноль. | `LOOP label` |
| **Наследование кода** | Использование общих блоков кода в нескольких процедурах. | `jmp common_end` |

## Ход выполнения работы  

1. **Подготовка среды**  
   * Установить ассемблер NASM (Linux) или MASM (Windows).  
   * Создать проект: `lab11.asm` и файл `Makefile` (Linux) или `lab11.bat` (Windows).  

2. **Написание вспомогательных процедур**  
   * `PrintInt` – вывод целого числа в консоль.  
   * `ReadInt` – чтение целого числа с клавиатуры.  

   ```asm
   ; NASM syntax
   section .text
   global _start

   _start:
       ; пример чтения
       call ReadInt
       ; пример вывода
       call PrintInt
   ```  

3. **Разработка основной программы**  
   * Считать два целых числа A и B.  
   * Вычислить их сумму, разность, произведение и частное.  
   * Использовать условные переходы для проверки деления на ноль.  
   * Вызвать процедуры `PrintInt` для вывода результатов.  

   ```asm
   ; Проверка деления на ноль
   cmp ebx, 0
   je  error_div
   div ebx
   jmp continue
   error_div:
       ; вывести сообщение об ошибке
   continue:
   ```  

4. **Отладка**  
   * Запустить программу в GDB (Linux) или в отладчике MASM.  
   * Установить точки останова на переходах.  
   * Проверить изменение флагов после каждой инструкции `cmp`.  

5. **Проверка корректности**  
   * Сравнить полученные результаты с калькулятором.  
   * Проверить, что при делении на ноль выводится сообщение.  

6. **Формирование отчёта**  
   * Вставить исходный код с комментариями.  
   * Приложить скриншоты отладчика.  
   * Сделать выводы о работе команд переходов.  

## Контрольные вопросы / задания для самопроверки  

1. Что происходит в регистре EIP после выполнения инструкции `CALL`?  
2. Какой флаг используется в условном переходе `JZ`?  
3. В чем разница между `JMP` и `LOOP`?  
4. Как передаются параметры в процедуру, если используется стек?  
5. Почему после `RET` регистр EIP получает значение из вершины стека?  
6. Что произойдет, если после `CALL` забыть выполнить `RET`?  
7. Как можно реализовать цикл `for` только с помощью команд переходов?  
8. Опишите порядок выполнения инструкций при условном переходе `JGE`.  
9. Как проверить, что стек не переполнен?  
10. Как вывести строку текста без использования системных вызовов в Linux?

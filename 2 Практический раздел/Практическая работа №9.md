# Практическая работа № 9  
Выполнение кодирования информации с помощью кода Хемминга и расчет значения корректирующего числа  

## Цель и задачи  
1. Познакомиться с принципами работы двоичного кода Хемминга и его применением в защите данных от ошибок.  
2. Научиться формировать кодовые слова, добавляя контрольные биты к исходному сообщению.  
3. Вычислять корректирующее число (сумму контрольных битов) и определять позицию возможной ошибки.  
4. Развивать навыки алгоритмического мышления и точности выполнения вычислений.  

## Теоретическая часть  
- **Бинарный код Хемминга** – линейный блочный код, который добавляет контрольные биты для обнаружения и исправления одиночных ошибок.  
- **Позиции битов** – нумеруются с единицы, контрольные биты находятся в позициях, которые являются степенями двойки (1, 2, 4, 8,…).  
- **Контрольные биты** – каждый контрольный бит \(c_k\) рассчитывается как XOR‑сумма всех битов, номера которых в двоичной записи имеют единицу в \(k\)-й позиции.  
- **Синдикат (корректирующее число)** – число, полученное из контрольных битов после чтения кода. Если синдикат равен нулю, ошибок нет.  
- **Кодовое слово** – исходное сообщение + контрольные биты, имеющее длину \(n = m + r\), где \(m\) – число данных битов, \(r\) – число контрольных.  
- **Условие полноты** – \(2^r \ge m + r + 1\).  
- **Коррекция ошибки** – если синдикат \(s \neq 0\), позиция \(s\) указывает на бит, который следует инвертировать.  

## Ход выполнения работы  

1. **Подготовка исходного сообщения**  
   - Выберите 4‑битное сообщение, например `1011`.  
   - Запишите его в массив: `m = [1,0,1,1]`.  

2. **Определение числа контрольных битов**  
   - Найдите минимальное \(r\), удовлетворяющее условию полноты:  
     `2^r >= 4 + r + 1`.  
     Для \(r=3\): `8 >= 8` – подходит.  
   - Итоговая длина кода \(n = 4 + 3 = 7\).  

3. **Формирование кодового слова**  
   - Вставьте контрольные позиции (1, 2, 4):  
     ```
     1 2 3 4 5 6 7
     c1 c2 m1 c3 m2 m3 m4
     ```
   - Заполните известные биты:  
     `c1 ?`, `c2 ?`, `c3 ?`, `m1=1`, `m2=0`, `m3=1`, `m4=1`.  

4. **Вычисление контрольных битов**  
   - Для \(c1\) (позиция 1): берём биты с номерами, где младший бит 1: 1, 3, 5, 7 → `? ,1,0,1`.  
     `c1 = XOR(?,1,0,1)`.  
   - Для \(c2\) (позиция 2): берём биты с номерами, где второй бит 1: 2, 3, 6, 7 → `?,1,1,1`.  
     `c2 = XOR(?,1,1,1)`.  
   - Для \(c3\) (позиция 4): берём биты с номерами, где третий бит 1: 4, 5, 6, 7 → `?,0,1,1`.  
     `c3 = XOR(?,0,1,1)`.  
   - Поскольку контрольные биты зависят от себя, решаем систему:  
     - `c1 = 1 ⊕ 0 ⊕ 1 = 0`  
     - `c2 = 1 ⊕ 1 ⊕ 1 = 1`  
     - `c3 = 0 ⊕ 1 ⊕ 1 = 0`  
   - Итоговый код: `0 1 1 0 0 1 1`.  

5. **Введение ошибки (для практики)**  
   - Сменим бит 5 (позицию 5) с `0` на `1`: `0 1 1 0 1 1 1`.  

6. **Расчёт синдиката**  
   - Повторяем расчёт контрольных битов для полученного слова:  
     - `c1' = 0 ⊕ 1 ⊕ 1 = 0`  
     - `c2' = 1 ⊕ 1 ⊕ 1 = 1`  
     - `c3' = 0 ⊕ 1 ⊕ 1 = 0`  
   - Синдикат `s = (c3'c2'c1') = 010₂ = 2₁₀`.  
   - Позиция `2` указывает, что ошибка находится во втором бите кода (позиция 2).  

7. **Коррекция ошибки**  
   - Инвертируем бит 2: `0 0 1 0 1 1 1`.  
   - Получаем исправленное слово: `0 0 1 0 1 1 1`.  

8. **Проверка**  
   - Пересчитаем контрольные биты для исправленного слова, убедимся, что синдикат равен нулю.  

## Контрольные вопросы / задания для самопроверки  

1. Как определяется число контрольных битов в коде Хемминга для сообщения из 8 битов?  
2. Опишите, как вычисляется контрольный бит \(c_2\) для позиции 2.  
3. Если после передачи кодовое слово `1 0 0 1 1 0 1` имеет синдикат `5`, какая позиция содержит ошибку?  
4. Почему условие \(2^r \ge m + r + 1\) необходимо для кода Хемминга?  
5. Напишите алгоритм, который принимает кодовое слово и возвращает исправленное слово.  
6. Как бы изменилась структура кода, если бы нужно было исправлять двойные ошибки?

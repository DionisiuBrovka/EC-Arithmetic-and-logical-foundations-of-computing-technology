# Лабораторная работа № 13  
Разработка и отладка программ с использованием подпрограмм и команд работы со стеком  

## Цель и задачи  
1. Освоить принципы работы стека в процессоре и понять его роль в реализации подпрограмм.  
2. Научиться писать и вызывать подпрограммы (функции) в ассемблере, использовать аргументы и возвращаемые значения.  
3. Научиться использовать команды управления стеком (`PUSH`, `POP`, `CALL`, `RET`) для реализации простых алгоритмов.  
4. Научиться отлаживать ассемблерный код с помощью симулятора и анализировать состояние стека и регистров.  

## Теоретическая часть  
| Понятие | Описание | Формула / Команда |
|--------|----------|-------------------|
| **Стек** | Сегмент памяти, реализующий структуру «последний пришёл – первый ушёл» (LIFO). | – |
| **TOP** | Указатель на вершину стека. | `ESP` (x86) |
| **CALL** | Вызов подпрограммы: сохраняет адрес возврата на стек и переходит к метке. | `CALL label` |
| **RET** | Возврат из подпрограммы: берёт адрес возврата из стека и переходит по нему. | `RET` |
| **PUSH** | Сохраняет регистр или значение на вершину стека. | `PUSH reg` |
| **POP** | Вытаскивает значение из вершины стека в регистр. | `POP reg` |
| **Subroutine** | Блок кода, выполняемый по вызову, может иметь собственные локальные переменные в стеке. | – |
| **Функция** | Подпрограмма, возвращающая значение через регистр (чаще `EAX`). | – |
| **Параметры функции** | Передаются через стек (первый аргумент – самый низ стек). | – |
| **Return value** | Возвращается в регистре `EAX`. | – |

## Ход выполнения работы  

1. **Подготовка среды**  
   - Установить симулятор ассемблера (NASM + DOSBox, MASM + EMU8086, или онлайн‑симулятор).  
   - Создать новый проект и файл `lab13.asm`.  

2. **Написание заглушек подпрограмм**  
   ```asm
   section .text
       global _start

   _start:
       ; вызов функции add
       push 5          ; второй аргумент
       push 10         ; первый аргумент
       call add
       add esp, 8      ; очистка аргументов
       ; результат в eax
       ; завершение программы
       mov eax, 1
       int 0x80

   ; ------------------------------------
   ; Функция add: возвращает сумму двух чисел
   add:
       push ebp
       mov ebp, esp
       mov eax, [ebp+8]    ; первый аргумент
       add eax, [ebp+12]   ; второй аргумент
       pop ebp
       ret
   ```  
   - Пояснение: `push` кладёт аргументы, `CALL` сохраняет адрес возврата, `RET` возвращает управление.  

3. **Использование стека для локальных переменных**  
   ```asm
   ; Функция mul: умножает два числа, используя локальную переменную
   mul:
       push ebp
       mov ebp, esp
       mov eax, [ebp+8]     ; первый аргумент
       mov edx, [ebp+12]    ; второй аргумент
       mov ecx, 0
   mul_loop:
       test edx, edx
       jz mul_end
       add ecx, eax
       dec edx
       jmp mul_loop
   mul_end:
       mov eax, ecx
       pop ebp
       ret
   ```  
   - Здесь `ecx` выступает как локальная переменная, хранится в регистре, но можно хранить в стеке (`push ecx`).  

4. **Отладка**  
   - Запустить симулятор, поставить точки останова на `CALL add` и `RET`.  
   - Проверить значения регистров `ESP`, `EBP`, `EAX` после каждого шага.  
   - Убедиться, что стек очищается после возврата (`ESP` возвращается к начальному значению).  

5. **Тестирование**  
   - Вызвать функции `add`, `mul`, `sub`, `div` с разными аргументами, записать результаты в память и вывести их через системный вызов `write`.  
   - Сравнить результаты с калькулятором.  

## Контрольные вопросы / задания для самопроверки  

1. Что хранится в регистре `ESP` и как он изменяется при выполнении `PUSH` и `POP`?  
2. Как передаются аргументы в функции, написанные в 32‑битном ассемблере?  
3. Почему в начале каждой подпрограммы используется `push ebp / mov ebp, esp`?  
4. Как реализовать возврат значения из подпрограммы в регистр `EAX`?  
5. Что произойдёт, если не очистить стек после вызова функции?  
6. Напишите подпрограмму, вычисляющую факториал числа, используя стек.  
7. Как проверить, что стек не переполнен во время работы программы?  
8. В чем разница между `CALL` и `JMP`?  
9. Почему важно сохранять `EBP` в начале подпрограммы?  
10. Как использовать команды `LEA` и `MOV` для доступа к аргументам функции?

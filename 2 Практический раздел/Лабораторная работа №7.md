# Лабораторная работа № 7  
**Исследование работы счетчиков экспериментальным путем**

## Цель и задачи
1. Познакомиться с принципами работы цифровых счетчиков в микросхемах и микропроцессорной архитектуре.  
2. Научиться измерять частоту и время перехода состояний счетчика при различных входных условиях.  
3. Оценить влияние параметров тактовой частоты и начального состояния на выходные сигналы.  
4. Развить навыки анализа экспериментальных данных и представления результатов в отчёте.

## Теоретическая часть
| Понятие | Описание | Формула / характеристика |
|---------|----------|--------------------------|
| **Счетчик** | Цифровое устройство, увеличивающее (или уменьшающее) значение при каждом импульсе тактового сигнала. | `Q_{n+1} = f(Q_n, T)` |
| **Такт** | Синхронный импульс, управляющий переходами состояний. | `f = 1/T_{т}` |
| **Счетчик до‑тч** | Счетчик, который при достижении предельного значения (тч) возвращается к нулю. | `Q_{max} = 2^n - 1` |
| **Счетчик до‑счет** | Счетчик, который при достижении предельного значения останавливается. | `Q_{max} = 2^n - 1` |
| **Счётчик с флагом** | Добавляет логический выход, сигнализирующий о достижении максимального значения. | `FLAG = (Q == Q_{max})` |
| **Период перехода** | Время, необходимое для перехода от одного состояния к другому. | `T_p = k·T_{т}` (k – число тактов) |
| **Скорость счёта** | Число переходов за секунду. | `S = 1/T_p` |
| **Системы счёта** | Десятичные, двоичные, двоично-десятичные, гексадецимальные. | – |
| **Код двоично-десятичный (BCD)** | Кодировка, где каждая цифра десятичного числа представлена четырьмя битами. | – |
| **Код Gray** | Код, в котором последовательные значения отличаются только одним битом. | `G = B ⊕ (B >> 1)` |

## Ход выполнения работы
1. **Подготовка оборудования**  
   - Подключите микросхему счетчика (например, 74LS161) к плате с тактовым генератором и осциллографом.  
   - Установите частоту такта в диапазоне 1 kHz–10 kHz.  
   - *Пример конфигурации*:  
     ```text
     Такт (CLK) ──┬───────► 74LS161
                  │
                  └─► Осциллограф (канал 1)
     ```

2. **Инициализация счетчика**  
   - Сбросьте счетчик до нуля (подача сигнала RESET).  
   - Установите начальное значение через входы D0–Dn.  
   - *Код для микроконтроллера (Arduino)*  
     ```cpp
     pinMode(RESET, OUTPUT);
     digitalWrite(RESET, LOW);   // сброс
     ```

3. **Запуск счёта и измерения**  
   - Включите тактовый сигнал.  
   - С помощью осциллографа зафиксируйте график выходных сигналов Q0–Qn.  
   - Измерьте период перехода `T_p` для каждого бита.  
   - *Пояснение*: при 1 kHz такт длится 1 мс, если каждый бит меняется за 4 такта, то `T_p = 4 мс`.

4. **Изменение частоты такта**  
   - Повторите измерения при 5 kHz и 10 kHz.  
   - Сравните полученные периоды перехода и скоростями счёта.  
   - *Пример таблицы*:  

     | Частота такта (kHz) | Период перехода (мс) | Скорость счёта (Hz) |
     |---------------------|----------------------|---------------------|
     | 1                   | 4                    | 250                 |
     | 5                   | 0,8                  | 1250                |
     | 10                  | 0,4                  | 2500                |

5. **Изучение влияния начального состояния**  
   - Запустите счёт с разными начальными значениями (0, 5, 10).  
   - Зафиксируйте, как меняется время, необходимое для достижения `Q_max`.  

6. **Анализ результатов**  
   - Составьте графики зависимости `T_p` от частоты такта.  
   - Оцените линейность и возможные отклонения.  

7. **Составление отчёта**  
   - Перейдите к разделу «Отчёт по работе» ниже.

## Контрольные вопросы / задания для самопроверки
1. Что такое счетчик до‑тч и до‑счет?  
2. Как вычислить период перехода `T_p` при заданной частоте такта?  
3. Каково влияние частоты такта на скорость счёта?  
4. Опишите процедуру сброса счетчика.  
5. Как изменить начальное состояние счётчика?  
6. Что происходит, если частота такта превышает допустимое значение для микросхемы?  
7. Как измерить период перехода на осциллографе?  
8. Сформулируйте формулу для кода Gray из двоичного числа.  
9. Почему важно, чтобы каждый бит менялся за фиксированное число тактов?  
10. Какой вывод можно сделать из графика зависимости `T_p` от частоты такта?

## Отчёт по работе
Структура отчёта:
1. **Титульный лист** – название работы, ФИО студента, группа, дата выполнения.  
2. **Введение** – цель лабораторной работы, краткое описание исследуемого объекта.  
3. **Теоретическая часть** – изложение ключевых понятий (см. раздел выше).  
4. **Методика исследования** – подробный ход выполнения, схемы подключения, параметры измерений.  
5. **Результаты** – таблицы, графики, измеренные значения `T_p`, скорость счёта, влияние начального состояния.  
6. **Обсуждение** – анализ полученных данных, сравнение с теоретическими ожиданиями, возможные ошибки.  
7. **Выводы** – основные выводы, ответы на контрольные вопросы.  
8. **Список использованных источников** – рекомендуемая литература.  
9. **Приложения** – схемы соединений, скриншоты осциллографа, код микроконтроллера (если применимо).

## Критерии оценки
1. Полнота и корректность выполнения всех шагов лабораторной работы.  
2. Точность измерений и правильность представления данных в таблицах/графиках.  
3. Глубина анализа результатов и обоснованные выводы.  
4. Чёткая и логичная структура отчёта, соблюдение требований к оформлению.  
5. Оформление и качество приложений (схемы, скриншоты, код).

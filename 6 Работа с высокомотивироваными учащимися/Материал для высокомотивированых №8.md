# Основы теории информации

Теория информации — фундаментальная дисциплина, связывающая математику, информатику и физику. В контексте арифметико‑логических основ вычислительной техники она предоставляет инструменты для оценки эффективности кодирования, анализа сложности алгоритмов, проектирования цифровых систем и безопасности. Понимание энтропии, кода и информации позволяет студентам создавать более компактные, надёжные и защищённые решения.

## Цели и задачи

1. **Ознакомить** студентов с ключевыми понятиями и формулами теории информации.
2. **Развить** аналитическое мышление через оценку эффективности кодов и передачу данных.
3. **Связать** теорию с практикой: показать, как энтропия и коды применяются в архитектуре ЭВМ, ассемблере и современных протоколах.
4. **Стимулировать** исследовательскую активность, задавая вопросы о новых кодах, компрессии и криптографии.
5. **Построить** фундамент для дальнейшего изучения теории алгоритмов, кодирования и безопасности.

## Ключевые понятия

| Термин | Расширенное определение |
|--------|-------------------------|
| **Энтропия (H)** | Мера неопределённости случайного события; в цифровой логике она показывает минимальное число битов, необходимое для представления исхода. |
| **Информация (I)** | Разница между энтропией до и после наблюдения; в вычислительной технике это количественная оценка полезных данных. |
| **Код Хаффмана** | Алгоритмический способ создания оптимального (суммарно минимального) двоичного кода для набора символов с разной частотой. |
| **Код Хэмминга** | Протокол обнаружения и исправления ошибок, использующий дополнительный бит для проверки целостности. |
| **Код База‑64** | Способ представления бинарных данных в ASCII, применяемый в протоколах передачи и кодировании файлов. |
| **Кодирование без потерь** | Трансформация данных, позволяющая восстановить оригинал без ошибок. |
| **Кодирование с потерями** | Метод, при котором часть информации теряется, но сохраняется достаточная структура для восприятия (например, JPEG). |
| **Кодирование в частотном домене** | Применение преобразования Фурье или DCT для более эффективной компрессии. |
| **Кодирование в пространственном домене** | Применение структурных свойств данных (например, корреляции) для снижения избыточности. |
| **Криптографический шум** | Случайные данные, вводимые в шифрование для обеспечения стойкости к атакам. |

### Исследовательские вставки

- **Вопрос 1**: Сгенерируйте случайный набор из 256 символов с различной частотой. Постройте код Хаффмана и вычислите среднюю длину кода. Сравните с фиксированной длиной 8 бит.
- **Вопрос 2**: Создайте 16‑битное слово, инжектируя одну ошибку. Используйте код Хэмминга для её обнаружения и исправления. Какова вероятность ошибки после исправления?

## Примеры и иллюстрации

```assembly
; Пример кода Хэмминга (12,8) в ассемблере x86
section .data
    data_byte db 0b10101100       ; исходный 8‑битный байт
    parity_bits db 0              ; место для 4 паритетных битов

section .text
    mov al, [data_byte]
    ; расчёт паритетных битов (простая реализация)
    mov bl, al
    shr bl, 1
    xor al, bl
    mov bl, al
    shr bl, 2
    xor al, bl
    mov [parity_bits], al
```

> **Кейс из индустрии**:  
> В современных SSD используются коды LDPC (Low‑Density Parity‑Check) для коррекции ошибок, позволяющие хранить более 10 % дополнительной информации без потерь.

## Проектные идеи

1. **Разработка собственного кодера Хаффмана** в ассемблере для микроконтроллера, измерение экономии памяти.
2. **Исследование эффективности LDPC‑кодов** в современных SSD, сравнение с традиционными кодами Хэмминга.
3. **Создание символьного компрессора** на основе энтропийного кодирования для текстовых файлов в ОС Linux.

## Выводы

Теория информации предоставляет количественные методы оценки и оптимизации данных. Понимание энтропии, кодов и ошибок открывает путь к более эффективным архитектурам памяти, надёжным системам передачи и безопасной криптографии. Эти знания фундаментальны для любого специалиста в области вычислительной техники, открывая перспективы в исследовании и практической разработке.

## Вопросы для самопроверки

1. Почему энтропия измеряется в битах, а не в других единицах измерения информации?
2. Как код Хаффмана обеспечивает минимальный средний кодовый слов для заданного распределения символов?
3. Почему код Хэмминга ограничен до исправления только одной ошибки в фиксированной длине слова?
4. В чем преимущество кодов LDPC над кодами Хэмминга в современных системах хранения данных?
5. Как можно использовать энтропию для оценки эффективности протокола сжатия изображений (например, JPEG)?
6. Почему при передаче данных через шумный канал добавление случайного шума может повысить устойчивость к атакам?
7. Как сравнить эффективность кодов без потерь и с потерями при заданной скорости передачи и уровне качества?
8. Какие ограничения существуют при применении кода Хаффмана в реальном времени (например, в потоковой передаче)?
9. Как можно применить теорию информации для оптимизации инструкций ассемблера, уменьшая количество битов в коде?
10. В чем разница между энтропией и информационным потоком в контексте сетевых протоколов?

# Основные архитектурно-технические решения современных микропроцессоров

Введение  
Современные микропроцессоры – ядро любой вычислительной системы. Их архитектура определяет эффективность выполнения арифметико‑логических операций, взаимодействие с памятью и периферией, а также возможности масштабирования и энергоэффективности. Знание ключевых решений в проектировании процессоров открывает двери к разработке высокопроизводительных систем, кибербезопасности, встроенных устройств и многопоточных приложений.  

## Цели и задачи

1. **Понять** фундаментальные архитектурные концепции (пipelining, superscalar, out‑of‑order execution, SIMD, VLIW).  
2. **Разобрать** современные микроархитектурные паттерны (ARMv8‑A, x86‑64, RISC‑V, GPU‑архитектуры).  
3. **Развить** аналитическое мышление при оценке компромиссов между производительностью, энергопотреблением и стоимостью.  
4. **Сформировать** навыки критического анализа технической документации и схем.  
5. **Подготовить** основы для самостоятельных исследований и проектной работы в области аппаратного обеспечения.

## Ключевые понятия

| Термин | Расширенное определение |
|--------|------------------------|
| **Пайплайн (pipeline)** | Последовательность этапов, через которые проходит инструкция, позволяющая обрабатывать несколько инструкций одновременно. Необходим для увеличения тактовой частоты и пропускной способности. |
| **Superscalar** | Архитектура, способная декодировать и выполнять более одной инструкции за такт. Требует сложных механизмов распределения ресурсов и прогнозирования ветвлений. |
| **Out‑of‑order execution** | Выполнение инструкций в порядке, отличном от их появления, для минимизации задержек из‑за зависимостей. Важен для эффективного использования ресурсов ALU и кэша. |
| **SIMD (Single Instruction, Multiple Data)** | Возможность выполнения одной инструкции над несколькими элементами данных параллельно, критична для мультимедийных и научных задач. |
| **Cache hierarchy** | Многоуровневая система кэшей (L1, L2, L3), уменьшающая латентность доступа к памяти. Проектирование кэша напрямую влияет на общую производительность. |
| **Branch predictor** | Алгоритм, предсказывающий направление ветвлений, чтобы избежать задержек из‑за отката пайплайна. |
| **Power gating** | Техника отключения питания неиспользуемых блоков для экономии энергии. |
| **Micro-op cache** | Кэш, хранящий разложенные на микрооперации инструкции, ускоряет декодирование. |
| **VLIW (Very Long Instruction Word)** | Архитектура, в которой одна инструкция содержит несколько параллельных микроопераций. Требует статического распределения ресурсов. |
| **Hardware multithreading** | Возможность одновременного выполнения нескольких потоков в одном ядре, повышая utilization. |

## Примеры и иллюстрации

### 1. Пайплайн x86‑64 (Intel Skylake)

```text
Stage 0: Fetch  |  Stage 1: Decode  |  Stage 2: Execute  |  Stage 3: Memory  |  Stage 4: Writeback
```

*Схема показывает 5‑ступенчатый пайплайн, где каждая инструкция может находиться в разных стадиях.*

### 2. SIMD‑операция на ARM Neon

```c
// Умножение двух массивов
void mul_arrays(float *a, float *b, float *c, int n) {
    for (int i = 0; i < n; i += 4) {
        float32x4_t va = vld1q_f32(&a[i]);
        float32x4_t vb = vld1q_f32(&b[i]);
        float32x4_t vc = vmulq_f32(va, vb);
        vst1q_f32(&c[i], vc);
    }
}
```

*Показано использование 128‑битных регистров Neon для параллельного умножения.*

### 3. GPU‑SIMT на NVIDIA CUDA

```cuda
__global__ void add(int *a, int *b, int *c, int n) {
    int idx = blockIdx.x * blockDim.x + threadIdx.x;
    if (idx < n) c[idx] = a[idx] + b[idx];
}
```

*Горизонтальная параллельность достигается через большое количество потоков в одном warp.*

### 4. Тест Spectre на Raspberry Pi 4

```bash
git clone https://github.com/google/spectre
cd spectre
make
./spectre
```

*Результат: вывод уязвимости в режиме user‑space.*
